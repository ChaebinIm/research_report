## ⓪ Executive Summary — 30초 요약

### 한 줄 정의
**차세대 메모리(Next-Gen Memory, NGM)** = DRAM·NAND처럼 "주류 메모리"가 되지 못한 채 변두리에서 살아남았으나, **AI·차량·우주·국방** 4대 메가트렌드에 힘입어 2026-2030년 사이 본격 폭발할 것으로 전망되는 8-10종의 신형 메모리 묶음.

### 핵심 명제 5가지
1. **DRAM·NAND·HBM** 3대 주류는 향후 5-10년 더 시장을 지배하지만, **틈새는 빠르게 NGM이 잠식**한다.
2. **MRAM**(자석 기반)이 차량·국방·산업 임베디드에서 **가장 먼저 상업적 변곡점**(2026-27)을 통과한다.
3. **CXL**(메모리 풀링)은 AI 데이터센터의 **두 번째 메모리 혁명**(첫 번째는 HBM). 2026년부터 본격 매출 인식, 2027-28 폭발.
4. **PIM**(메모리 안 계산)은 **폰노이만 병목**의 궁극 해결책. HBM4부터 logic die 통합 시작.
5. **HBF**(High Bandwidth Flash)는 SK하이닉스·Sandisk 합작으로 2026.02 표준화 착수, **2027 첫 AI 추론용 샘플 출하 예정** — HBM 보완재로 부상.

## ① 왜 차세대 메모리인가 — 5분 입문

### 1.1 메모리란 무엇인가 — "컴퓨터의 단기 기억"

**비유**: 사람이 책상에서 일할 때를 떠올려보자.

- **CPU/GPU = 두뇌** (계산하는 부분)
- **DRAM = 책상 위** (지금 보고 있는 책·노트, 빨리 꺼내 쓸 수 있지만 자리 비좁음)
- **NAND/SSD = 책장** (많이 쌓아두지만 가져오는 데 시간 걸림)
- **HDD = 창고** (옛날 자료, 거의 안 씀)

| 종류 | 일상 비유 | 속도 | 용량 | 가격 |
|---|---|---|---|---|
| CPU 캐시(SRAM) | 손에 든 메모지 | 매우 빠름 | 매우 작음(MB) | 매우 비쌈 |
| DRAM | 책상 위 | 빠름 | 보통(GB) | 보통 |
| NAND/SSD | 책장 | 느림 | 큼(TB) | 저렴 |
| HDD | 창고 | 매우 느림 | 매우 큼 | 매우 저렴 |

**핵심 트레이드오프**: 빠르면 비싸고 작다 / 느리면 싸고 크다. 이 사이를 메우는 게 **차세대 메모리**의 역할.

### 1.2 기존 메모리의 한계 4가지

#### (1) DRAM의 한계 — **휘발성**과 **전력 소모**

> **휘발성(Volatile)** = 전원 끄면 데이터 사라짐. 마치 화이트보드처럼.

DRAM은 1초에 수만 번씩 데이터를 다시 써줘야(refresh) 한다. 이걸 **누설 전류** 때문이라 부르는데, 데이터센터 전기료의 **20-30%가 DRAM refresh**에 쓰인다. AI 서버에서는 더 심각.

#### (2) NAND의 한계 — **쓰기 수명**과 **속도**

NAND는 비휘발성(전원 꺼도 유지)이지만:
- 쓰기 수명: **3D TLC 1,000-3,000회**, QLC는 100-500회 → 같은 셀에 무한 쓸 수 없음
- 쓰기 속도: DRAM의 1/100-1/1000

**비유**: NAND는 연필로 종이에 쓰는 것. 지웠다 다시 쓰기 반복하면 종이가 닳는다.

#### (3) SRAM의 한계 — **비쌈**, **저용량**

CPU 캐시로 쓰이는 SRAM은 **6개 트랜지스터**로 1비트 저장 → DRAM(1T1C)의 6배 자리 차지. 그래서 32MB 정도만 칩에 박을 수 있음. **칩 면적의 30-50%를 SRAM이 차지**하는데도 부족.

#### (4) NOR Flash의 한계 — **저밀도**

NAND 대비 **속도는 빠르나 용량 1/100**. MCU에 코드 저장용으로만 쓰임. 차량용 MCU는 점점 더 큰 코드를 요구 → NOR로 부족.

### 1.3 폰노이만 병목 — "두뇌와 책상 사이가 너무 멀다"

**폰노이만 구조(Von Neumann Architecture)**: 1945년 폰노이만이 설계한 컴퓨터 구조. CPU와 메모리가 **분리**되어 있고, 데이터를 주고받음.

> **폰노이만 병목**: CPU는 1초에 수십억 번 연산하는데, 메모리에서 데이터 한 번 가져오는 데 수백 사이클 낭비. 마치 천재 두뇌인데 책상이 10m 떨어져 있어 매번 걸어가야 하는 꼴.

**AI에서는 더 심각**: ChatGPT 같은 LLM은 **메모리에서 데이터 읽는 시간이 전체 추론 시간의 70-80%**. 실제 GPU 연산 시간은 20-30%뿐. → **메모리 병목 = AI 시대의 가장 비싼 문제**.

해결책 후보:
- **HBM**: 메모리를 GPU 옆에 적층 → 거리 단축 (이미 상용화)
- **CXL**: 메모리를 풀링 → 더 큰 책상 마련 (2025-)
- **PIM**: 메모리 안에서 직접 계산 → 책상이 두뇌 (2026-)
- **HBF**: NAND를 HBM처럼 적층 → 더 큰 캐시 (2027-)

### 1.4 새로운 수요 4가지

| 분야 | 요구 | 기존 메모리 한계 | NGM 해법 |
|---|---|---|---|
| **AI** | 거대 모델 가중치 캐싱 | HBM 비쌈·작음 | HBF·CXL·PIM |
| **차량** | 신뢰성·고온·즉시 부팅 | NAND 마모, NOR 작음 | eMRAM·eFRAM |
| **우주·국방** | 방사선 저항·내구성 | DRAM 단일 이벤트 오류 | MRAM(방사선 강함) |
| **IoT/웨어러블** | 저전력·즉시 깨어남 | NOR/NAND 전력 큼 | MRAM·FeRAM |

### 1.5 차세대 메모리의 공통 키워드 — **NVRAM** = 비휘발성 RAM

> **NVRAM(Non-Volatile RAM)** = 전원 꺼도 데이터 유지되는 RAM. 즉 "DRAM처럼 빠르면서도 NAND처럼 안 사라지는" 꿈의 메모리.

MRAM·ReRAM·PCM·FeRAM 모두 NVRAM 카테고리. 각각 0/1을 저장하는 **물리 원리가 다를 뿐**.

## ③ 차세대 메모리 5종 — 짧은 입문 (단독 보고서로 위임)

각 기술별 단독 심층 보고서가 별도 발주됨. 본 챕터는 핵심만 1-2페이지씩.

### 3.1 MRAM — 자석 N/S로 저장 ("나침반 비유")

#### 비유
한 셀 = 작은 나침반. **N극이 위면 0, S극이 위면 1**. 자석은 전원 꺼도 방향 유지 → 비휘발성. 외부 전류로 N/S 방향 뒤집어 쓰기.

#### 구조 — MTJ(Magnetic Tunnel Junction, 자기터널접합)

```
[고정층 자석 ↑] ← N/S 고정
[얇은 절연막]   ← 1nm 산화물
[자유층 자석 ↑↓]← 쓰기로 방향 바뀜
```

- 두 자석 방향 같음(평행) = 저항 낮음 = "0"
- 두 자석 방향 반대(반평행) = 저항 높음 = "1"

#### 종류
- **STT-MRAM**: Spin-Transfer Torque. 전류로 자석 회전. 현재 주류.
- **SOT-MRAM**: Spin-Orbit Torque. 더 빠름·저전력. 차세대.
- **VCMA-MRAM**: 전압 기반. 연구 단계.

#### 강점·약점
| 강점 | 약점 |
|---|---|
| 비휘발 + 쓰기수명 ∞에 가까움 | 단가 높음(DRAM의 100-1000배) |
| 방사선 저항 (우주·국방용) | 셀 면적 큼 (DRAM의 2-3배) |
| DRAM급 속도 | 자기 잡음 시 데이터 손상 가능 |

#### 주요 플레이어
- **Everspin**(MRAM·미국): MRAM 전문, 산업·국방, 4Mb-1Gb
- **삼성전자**: eMRAM 28nm FD-SOI 양산
- **TSMC**: eMRAM 22nm/16nm, NXP 협력
- **GlobalFoundries**: eMRAM 22FDX, Auto Grade 1, Bosch tape-out
- **SK하이닉스**: 연구 단계, 아직 양산 X
- **중국 신규 진입**: SMIC·HLMC가 2025-26 진입

#### 시장 (Yole 2026 추정)
- 2026: 약 $1.5B
- 2030: $4-6B (CAGR 25-30%)
- 키 드라이버: 차량 MCU + 데이터센터 캐시

> **단독 보고서**: `MRAM_단독분석.md` (별도 발주)

### 3.2 ReRAM — 저항 변화로 저장 ("필라멘트 형성")

#### 비유
한 셀 = 작은 도화선. 전압을 걸면 **금속 산화물 안에 미세 필라멘트가 형성**되어 전류가 흐름(저저항=1). 반대 전압으로 필라멘트 끊으면 전류 안 흐름(고저항=0).

#### 구조 — MIM(Metal-Insulator-Metal)

```
[금속 전극]      ← 상단
[금속 산화물]    ← HfO2, TaOx 등
[금속 전극]      ← 하단
```

매우 단순. NAND·DRAM 대비 **마스크 추가 거의 없이 28nm 공정에 얹을 수 있음** → 임베디드용으로 인기.

#### 강점·약점
| 강점 | 약점 |
|---|---|
| 단순 구조 → 저렴 | 셀별 균일성 낮음(소자별 편차) |
| 고밀도 (3D 적층 가능) | 쓰기 수명 10⁶-10⁹로 MRAM/FeRAM보다 짧음 |
| 뉴로모픽(아날로그 가변) 적합 | 양산 노하우 미성숙 |

#### 주요 플레이어
- **Crossbar**(미국·비상장): ReRAM 선구자
- **Weebit Nano**(호주·ASX:WBT): IP 라이선싱 모델 — DB하이텍·onsemi 협력. 2026 onsemi qualification 변곡점.
- **Adesto/Dialog/Renesas**(CBRAM): 차량 MCU
- **삼성전자**: 연구 단계
- **TSMC**: eReRAM 검토 중
- **GlobalFoundries**: eReRAM 22FDX 발표 (2024)
- **Macronix**(대만): 3D ReRAM 연구

#### 한국 관련
- **DB하이텍**이 Weebit Nano의 IP를 라이선싱. 28nm BCD 공정에 통합 → 차량용 MCU 노린다.

> **단독 보고서**: `ReRAM_단독분석.md` (별도 발주)

### 3.3 PCM — 결정·비결정 상태로 저장 ("아이스크림 녹았다 굳기")

#### 비유
**칼코게나이드**(주로 GST = Ge-Sb-Te)라는 물질이 셀 안에 들어있음. 전류로 가열하면:
- **빠르게 식음 → 비결정(아모르퍼스) → 저항 높음 → "0"**
- **천천히 식음 → 결정 → 저항 낮음 → "1"**

마치 아이스크림이 녹았다 굳을 때 결정이 생기는 것과 비슷.

#### 구조

```
[상단 전극]
[GST 물질] ← 가열-냉각으로 상 변화
[히터]    ← 전류로 가열
[하단 전극]
```

#### 강점·약점
| 강점 | 약점 |
|---|---|
| 다중레벨 셀(MLC) 가능 → 고밀도 | 쓰기 시 가열 → 전력 큼 |
| 빠른 쓰기 (100ns 수준) | 인접 셀 간섭 (열 누설) |
| SCM에 최적 | 양산 어려움 — Intel Optane 단종 사례 |

#### 역사 — Intel Optane의 부상과 좌절
- **2015 Intel·Micron 3D XPoint 발표** → DRAM-NAND 사이 SCM 개척
- **2017-2020 Intel Optane SSD/DCPMM 출시** → 데이터센터 일부 채택
- **2022 Micron 철수, Intel 단종** → 시장 작아 ROI 안 맞음
- **남은 자산**: Micron이 PCM IP 일부 판매, 중국·대만 업체 인수 시도

#### 주요 플레이어 (2026 현재)
- **STMicroelectronics**: ePCM 차량 MCU 양산 중 (Stellar, 28nm FD-SOI)
- **삼성전자**: PCM 연구 (2009 첫 양산 후 축소)
- **중국 GigaDevice·HLMC**: PCM 진입
- **YMTC**(중국 NAND): 3D Xtacking 일부 활용

> **단독 보고서**: `PCM_단독분석.md` (별도 발주)

### 3.4 FeRAM — 강유전체 분극으로 저장 ("도미노 한 줄")

#### 비유
**강유전체**(PZT 또는 HfO2)는 외부 전압을 끄고 나서도 **분극 방향(↑ 또는 ↓)을 유지**하는 특수 물질. 마치 도미노를 한 번 쓰러뜨리면 그 상태로 유지되는 것과 같음. 전압 방향만 바꾸면 즉시 뒤집힘.

#### 구조
```
[상단 전극]
[강유전체]  ← PZT 또는 HfO2
[하단 전극]
```

#### 강점·약점
| 강점 | 약점 |
|---|---|
| 쓰기 수명 10¹²-10¹⁴ (사실상 무한) | 셀 면적 매우 큼(고밀도 어려움) |
| 매우 저전력 (수십 nJ) | 단가 비쌈 |
| 고온(125°C+) 동작 가능 | 면적상 메인 메모리 부적합 |
| 100조 회 쓰기 → 데이터로깅 최적 | - |

#### 주요 플레이어
- **Infineon**(구 Cypress·Ramtron 인수): F-RAM 산업·차량 데이터로깅 1위. 100조 회 쓰기 보장.
- **Texas Instruments**: MSP430 FRAM MCU. 저전력 IoT 1위.
- **후지쯔(Fujitsu)**: PZT FRAM 산업용
- **Lapis Semiconductor**(일본): 산업용
- **삼성전자**: HfO2 FeFET 연구

#### HfO2 르네상스
2010년대까지 PZT(납·티타늄·지르코늄) 기반 FeRAM은 **CMOS 공정에 안 맞아** 30nm 이하로 못 내려감. 그러나 **2011년 HfO2 강유전성 발견** 이후 **22-28nm 공정과 통합 가능** → 임베디드 FRAM 부활.

GlobalFoundries·TSMC가 22FDX/16nm에 **eFRAM 옵션 추가**. 차량 MCU 후보로 부상.

> **단독 보고서**: `FeRAM_단독분석.md` (별도 발주)

### 3.5 NRAM·CBRAM·기타 (보조)

#### NRAM (Nantero, 탄소나노튜브 RAM)
- **원리**: 탄소나노튜브가 접촉 = 1, 분리 = 0
- **장점**: 매우 빠름 + 비휘발 + 고온 강함
- **상태**: Nantero 25년째 개발 중, 2024-2025 일부 양산 시도
- **파트너**: 후지쯔 등

#### CBRAM (Conductive Bridge RAM)
- **원리**: 금속 이온이 절연체 안에서 다리(브리지) 형성/파괴
- **종류**: ReRAM의 한 갈래로 분류됨
- **플레이어**: Adesto/Dialog/Renesas

#### MagRAM·OxRAM·기타
- **OxRAM** = ReRAM의 산화물 버전
- **VCMA-MRAM** = 전압 제어 MRAM (저전력 차세대)

> **이 보조 기술들은 본 보고서에서 별도 다루지 않음**. 관심 시 SemiWiki·EE Times·MRAM-Info 참조.

## ④ CXL — 메모리 풀링 혁명 (별도 챕터, 단독 보고서로 위임)

### 4.1 CXL이 무엇인가 — "공유 창고 메모리"

#### 비유
기존 서버: 각 GPU가 자기 옆에 작은 책상(HBM)을 두고 일함. 책상 부족하면 일 못 함.
CXL 서버: **모든 GPU·CPU가 큰 공용 창고(CXL 메모리 풀)**에 접근. 필요한 만큼만 가져옴.

#### 풀네임·정의
**CXL = Compute Express Link**. Intel이 2019년 발표한 **고속 인터커넥트 표준**. PCIe 5.0/6.0 위에서 동작.

#### 핵심 기능 3가지
1. **CXL.io** — PCIe와 호환 (기존 장치)
2. **CXL.cache** — CPU↔디바이스 간 캐시 일관성
3. **CXL.mem** — **메모리 풀링·확장의 핵심** ★

### 4.2 CXL 1.1 → 2.0 → 3.0 → 3.2 진화

| 세대 | 발표 | 핵심 기능 | 상용화 |
|---|---|---|---|
| **CXL 1.1** | 2019 | 메모리 확장(단일 호스트) | 2022-23 (제한적) |
| **CXL 2.0** | 2020 | **메모리 풀링**(다중 호스트) | 2024-25 (본격) |
| **CXL 3.0** | 2022 | 멀티 레벨 스위칭, P2P | 2025-26 |
| **CXL 3.1/3.2** | 2024-25 | 광 통신, 보안 강화 | 2026-27 |

#### CXL 2.0의 의미
**메모리를 풀링** = 여러 서버가 하나의 메모리 풀을 공유. 각 서버는 필요한 만큼만 할당받음. → 메모리 활용률 30% → 70-80%로 점프. **데이터센터 메모리 비용 30-40% 절감 효과**.

### 4.3 CXL 메모리 종류 — Type 1/2/3

| 타입 | 용도 | 예시 |
|---|---|---|
| **Type 1** | 가속기 (캐시 일관성만) | NIC, GPU |
| **Type 2** | 가속기 + 메모리 | AI 가속기 |
| **Type 3** | **메모리 확장 모듈** ★ | CXL DRAM 모듈 (CMM-D) |

> **CMM-D**(CXL Memory Module-DRAM) = SK하이닉스·삼성의 CXL DRAM 제품 브랜드.

### 4.4 주요 플레이어

#### 메모리 모듈 (Module 제조)
- **SK하이닉스**: **CMM-D 96GB·128GB·256GB**. 2026 Q1 양산 본격화.
- **삼성전자**: CMM-D + **Pangea v2/v3** 풀링 솔루션. 2026년 Pangea v3 (CXL 3.2 + 광통신) 발표 예정.
- **Micron**: CXL 모듈 출시 (CZ120 시리즈)

#### 컨트롤러 (메모리 컨트롤러 칩)
- **Astera Labs(ALAB)** ★: **Leo CXL Smart Memory Controller** — 시장 1위. 2025 OCP에서 풀링 데모, 3x throughput / 3x lower latency 입증.
- **Marvell(MRVL)**: Tanzanite Silicon Logic 인수. CXL 컨트롤러 2위 추격.
- **Montage Technology**(중국·688008.SH): CXL 컨트롤러
- **Rambus(RMBS)**: CXL 컨트롤러 + IP
- **Microchip(MCHP)**: SmartMemory CXL
- **XConn**: CXL 스위치 IC

#### 신규 스타트업
- **XCENA**(한국·SK하이닉스·삼성 출신): CXL + compute 통합

### 4.5 시장 규모
- 2025: ~$0.5B
- 2026: ~$2B
- 2028: ~$8B
- 2030: ~$15-20B (CAGR 100%+)

#### 폭발 시점
**2026년 본격, 2027-28 폭발**. NVIDIA Rubin/Rubin Ultra 아키텍처가 CXL 본격 채택 + AMD·Intel CPU가 CXL 2.0 표준화.

## ⑤ PIM — 폰노이만 한계 극복 (별도 챕터, 단독 보고서로 위임)

### 5.1 PIM 비유 — "메모리에서 직접 계산"

기존 컴퓨터:
```
GPU ←─ (긴 길) ─→ DRAM
       데이터를 매번 가져와야 함 (전기 80% 낭비)
```

PIM:
```
DRAM 안에 작은 계산기 내장
   ↓
결과만 GPU로 전달 (데이터 이동 90% 감소)
```

### 5.2 PIM 종류

| 종류 | 베이스 | 용도 | 회사 |
|---|---|---|---|
| **HBM-PIM** | HBM 안에 연산 유닛 | AI 가속기 | 삼성 (Aquabolt-XL) |
| **AiM** | DRAM 기반 PIM | 추천 시스템·LLM | SK하이닉스 |
| **LPDDR-PIM** | 모바일 DRAM PIM | 온디바이스 AI | 삼성 |
| **GDDR-PIM** | GDDR 기반 | GPU 보조 | 연구 |
| **CXL-PIM** | CXL 카드 + 컴퓨트 | 인메모리 DB | XCENA, UPMEM |

### 5.3 주요 플레이어

#### IDM
- **삼성전자**: HBM-PIM(Aquabolt-XL), LPDDR-PIM, AXDIMM
- **SK하이닉스**: AiM(GDDR6-AiM 베이스), AiMX 가속기 카드
- **Micron**: ATOMs(연구)

#### 스타트업
- **UPMEM**(프랑스): DPU(DRAM Processing Unit). DDR4 기반 PIM 모듈 출시.
- **Mythic**: 아날로그 PIM(머트릭스 곱셈 가속) — 2023 자금난, 부활 시도
- **Syntiant**: 음성 인식 엣지 PIM
- **GSI Technology(GSIT)**: APU(Associative Processing Unit) — 검색 가속

### 5.4 시장 규모 — 가장 불확실

PIM은 **표준 부재**가 최대 약점. 각사 독자 규격 → 소프트웨어 생태계 미성숙. 그러나 HBM4 부터 **logic die 통합** → PIM 표준화 가능성.

- 2026: $0.5-1B
- 2030: $5-20B (격차 매우 큼 — 스폐셜티 vs 메인스트림 전환 여부)

> **단독 보고서**: `PIM_단독분석.md` (별도 발주)

## ⑥ HBF (High Bandwidth Flash) — 신규 컨셉, 2026.02 표준화 착수

### 6.1 HBF의 정의 — "HBM처럼 NAND를 적층"

#### 비유
- **HBM** = DRAM을 12-16층 적층 → 빠르고 고대역 (그러나 작고 비쌈)
- **HBF** = **NAND를 동일 방식으로 적층** → 비휘발 + HBM 80% 속도 + **HBM 8-16배 용량** + 비슷한 단가

#### 핵심 스펙 (2026 사양 초안)
- **대역폭**: HBM3E의 **약 80%** (HBM3E 1.2 TB/s 기준 → 약 1 TB/s)
- **용량**: HBM 1개 스택의 **8-16배** (HBM3E 36GB → HBF 288-576GB)
- **단가**: HBM의 **1/4-1/8**
- **수명**: NAND 기반이므로 쓰기 수명 제한 → **읽기 위주 워크로드**(추론용)에 최적

### 6.2 왜 지금 등장하나 — AI 추론 시대

**훈련(Training)**: 데이터 빠르게 쓰고 읽기 → HBM 필요
**추론(Inference)**: 모델 가중치는 한 번 로드 후 **읽기만 무수히** → HBF 적합

> **2026 추세**: AI 워크로드가 훈련에서 **추론으로 무게중심 이동**. ChatGPT 일일 쿼리 100억 건 → 추론용 메모리 수요 폭증.

### 6.3 SK하이닉스 + Sandisk 합작

#### 타임라인
- **2025-08-06**: Sandisk·SK하이닉스 MOU 체결, HBF 표준화 협력 발표
- **2026-02-25**: **HBF Spec. Standardization Consortium Kick-Off** (Sandisk Milpitas 본사). OCP 워크스트림 신설.
- **2026 H2**: Sandisk 첫 HBF 샘플 출하 예정
- **2027 초**: 첫 HBF 탑재 AI 추론 디바이스 샘플
- **2028+**: 본격 양산

#### 기술 기반
- **Sandisk BiCS Flash** + **CBA(CMOS-Bonded Array) wafer bonding**
- **SK하이닉스 NAND 적층 기술** + HBM 패키징 노하우

### 6.4 시장 잠재력
TrendForce 추정: HBF 시장이 **2030년까지 폭발 성장**. AI 추론 시장의 일부를 HBM에서 빼앗음.

#### 경쟁 위협 vs 기회
- **HBM 입장**: HBF는 일부 추론 워크로드 잠식 → 위협
- **NAND 입장**: NAND가 **고부가가치 영역으로 진입** → 기회
- **Sandisk·SK하이닉스 입장**: 양사 모두 NAND·DRAM 모두 보유 → **윈윈**

#### 누락 플레이어
- **삼성전자**: HBF 별도 진입 가능성 (2026 H2 자체 발표 가능성)
- **Micron**: 후발 추격 가능성
- **YMTC**(중국): 자체 HBF 컨셉 추진 가능

## ⑦ 3D Stacked·하이브리드 메모리·Optical Memory

### 7.1 DRAM 12층 적층 — HBM4 시대 개막

#### HBM 적층 진화
| 세대 | 적층 | 대역폭/스택 | 용량/스택 | 양산 |
|---|---|---|---|---|
| HBM2 | 4-8 hi | 256 GB/s | 4-8GB | 2016-18 |
| HBM2E | 8-12 hi | 460 GB/s | 8-16GB | 2020-21 |
| HBM3 | 8-12 hi | 819 GB/s | 16-24GB | 2022-23 |
| HBM3E | 8-12 hi | 1.2 TB/s | 24-36GB | 2024-25 |
| **HBM4** | **12-16 hi** | **1.6 TB/s** | **36-48GB** | **2026** |
| **HBM4E** | 16+ hi | 2+ TB/s | 64GB+ | 2027-28 |

#### HBM4의 게임체인저 — Logic Base Die
HBM4부터 **밑에 logic die**(연산용 칩)를 깔음. 단순 메모리에서 **반쯤 컴퓨팅 디바이스**로 진화. PIM 통합의 출발점.

### 7.2 하이브리드 메모리 — DRAM + NVRAM 결합

**NVDIMM(Non-Volatile DIMM)**: DRAM + NAND 결합. 전원 끊기면 DRAM 내용을 NAND로 백업. 데이터센터 일부 사용.

### 10.1 AI 가속기 캐시 — HBM-PIM·CXL Memory (2027 시점 본격)

3. **HBF** (NAND 기반 대용량 캐시, 2027 샘플)
4. **HBM-PIM** (메모리 안 직접 연산)

#### 시간표

| 2026 | HBM4 + CXL 2.0 본격 + HBF 표준 |

### 10.2 차량용 MCU 임베디드 — eMRAM·eFRAM·eReRAM (2025-30 본격)

#### 차량 MCU의 진화
| 세대 | 시점 | 메모리 | 적용 |
|---|---|---|---|
| 1세대 | -2000 | NOR Flash 16-64KB | 단순 ECU |
| 2세대 | 2000-2010 | NOR 1-4MB | 다기능 ECU |
| 3세대 | 2010-2020 | NOR/eFlash 4-16MB | ADAS·인포테인먼트 |
| **4세대** | **2020-2030** | **eMRAM/eFRAM/ePCM 16-64MB** | **존 컨트롤러·SDV** |
| 5세대 | 2030+ | NGM 100MB+ + 작은 NAND | 자율주행 L4-5 |

#### 왜 NGM인가
- **NOR Flash 한계**: 28nm 이하 미세화 어려움 → 면적·단가 증가
- **NAND 한계**: 차량 고온·진동에 마모 빠름
- **NGM 장점**: 비휘발성 + 고온 동작 + 빠른 즉시 부팅 + 마모 없음

## ⑬ 9가지 핵심 동인 + 9대 리스크

### 13.1 9가지 핵심 동인 (성장 이유)

1. **AI 메모리 병목** — LLM 추론 시간의 70%+가 메모리 대기. NGM 수요 폭증.
2. **차량 SDV(Software-Defined Vehicle) 전환** — 존 컨트롤러 = 16-64MB 임베디드 메모리 필요. NOR 한계.
3. **HBM 단가 부담** — HBM3E $30/GB → AI 서버에 더 싼 대용량 캐시(HBF) 수요.
4. **CHIPS Act·EU Chips Act** — 미국·EU가 NGM 신산업 보조금 책정.
5. **CXL 표준 통일** — Intel·AMD·NVIDIA 모두 CXL 2.0 공식 채택 (2025-26).
6. **방사선 저항 MRAM의 우주·국방 채택** — 미·중 우주 경쟁 가속.
7. **HfO2 강유전체 발견** — FeRAM 미세화 가능 → 임베디드 부활.
8. **eMRAM 파운드리 옵션화** — TSMC·GF·삼성 모두 22-16nm eMRAM PDK 제공.
9. **데이터센터 전기료 폭증** — refresh 전력 절감 필요 → NVRAM 매력 증가.

### 13.2 9대 리스크 (조심할 점)

1. **DRAM·NAND 사이클 둔화** — NGM 시장은 작아 메모리 대형주에 가려질 수 있음.
2. **HBM이 모든 영역 잠식** — HBM4·5가 너무 강해 NGM 자리 없을 가능성.
3. **CXL 표준 분열** — 회사별 독자 규격 → 생태계 파편화.
4. **PIM 소프트웨어 미성숙** — 표준 라이브러리 부재. CUDA같은 생태계 부재.
5. **MRAM 단가 하락 정체** — DRAM 1/100까지 안 떨어지면 대체 어려움.
6. **차량 사이클 둔화** — 2024-25 차량 반도체 사이클 부진. NGM 채택 지연.
7. **중국 진입** — SMIC·HLMC·YMTC가 MRAM·ReRAM·PCM 진입. 가격 경쟁 가속.
8. **미·중 수출 통제** — NGM 장비·기술 수출 제한 가능성.
9. **Pure Play 종목 변동성** — Everspin·Weebit은 시총 작아 변동성 큼.

## ⑮-θ 부록 8 — NGM 채택 결정 프레임워크 (실무자용)

### H1. 시스템 설계자 관점 — 어느 메모리를 골라야 하나

#### 결정 트리 (단순화)

**Q1. 비휘발성 필요?**
- **NO** → DRAM (혹은 HBM)
- **YES** → Q2

**Q2. 쓰기 횟수?**
- **무한** (10⁹+) → MRAM 또는 FeRAM
- **중간** (10⁶-10⁹) → ReRAM 또는 PCM
- **저빈도** (10⁵-10⁶) → NAND 또는 NOR

**Q3. 속도?**
- **DRAM급(<50ns)** → MRAM
- **수백 ns** → PCM, ReRAM, FeRAM
- **수천 ns** → NAND

**Q4. 용량?**
- **GB-TB** → NAND, HBF, CXL Memory
- **MB-GB** → MRAM, PCM, DRAM
- **KB-MB** → FeRAM, eMRAM, NOR

**Q5. 단가 민감도?**
- **매우 민감** → NAND ($0.05/GB)
- **보통** → DRAM ($3/GB)
- **저비용 OK** → MRAM ($500-2000/GB)

### H2. 적용별 추천

#### IoT 센서 노드
- **MCU 코드**: TI MSP430 FRAM 또는 STM ePCM

## ⑮-ι 부록 9 — 차량용 NGM 심층 (eMRAM·ePCM·eFRAM)

### I1. 차량 메모리 진화 배경

#### SDV(Software-Defined Vehicle) 시대
- **2020-2025**: ECU 100개+ 분산 → 통합 어려움
- **2025-2030**: **존(Zone) 컨트롤러**로 통합. 1대당 4-8개 존, 각 존이 16-64MB 메모리 보유

#### 메모리 요구의 변화
| 시점 | MCU 메모리 | 메모리 종류 |
|---|---|---|
| 2010 | 1-4MB | NOR Flash |
| 2020 | 4-16MB | eFlash + DRAM |
| 2025 | 16-64MB | eFlash + DRAM + NGM 일부 |
| 2030 | 64-256MB | NGM 본격 + LPDDR + 일부 NAND |

### I2. NOR Flash의 한계 — 28nm 벽
NOR는 **고전압 쓰기**가 필요해 28nm 이하 미세화 어려움. 결과:
- 면적당 단가 올라감
- 28nm 노드 주변에서 NGM과 가격 교차

### I3. NGM 채택 매트릭스 (차량 MCU 기준)

| 회사 | 현재(2026) | 2027-28 | 2030 |
|---|---|---|---|
| **STM** | ePCM (Stellar) | + eMRAM 검토 | NGM 본격 |
| **NXP** | TSMC 16nm eMRAM | 본격 양산 | 차세대 노드 |
| **Infineon** | eFlash + F-RAM | + eMRAM 검토 | 다양화 |
| **Renesas** | eFlash + CBRAM | + eMRAM 본격 | 다양화 |
| **TI** | eFlash + FRAM (MSP430) | 확대 | 다양화 |
