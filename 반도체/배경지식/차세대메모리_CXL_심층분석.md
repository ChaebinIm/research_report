# 차세대 메모리 CXL(Compute Express Link) 단독 심층 분석 — 데이터센터의 새 동맥

> **작성일**: 2026-05-11
> **대상 독자**: 한국 개인 투자자 (문과·기술 배경지식 0)
> **분량**: 약 60,000자
> **핵심 메시지**: CXL은 "CPU·GPU·메모리를 PCIe보다 더 똑똑하게 묶어주는 새 도로". HBM이 단거리 고속도로라면 CXL은 도시 전체를 잇는 순환도로. 2024 약 5억 달러였던 시장이 2030년 150~200억 달러로 30배 폭발 예상. 한국 SK하이닉스·삼성이 모듈에서 1~2위, 컨트롤러는 미국 Astera Labs(ALAB)가 압도적 1위.

## 1. CXL 5분 입문 — "도서관 공유" 비유로 단숨에

### 1-1. 비유부터 — 책상 vs 도서관

옛날 사무실을 상상해보자. 각자 책상 위에 작은 책장(D램)을 두고, 자기가 자주 보는 책(데이터)만 거기 꽂아둔다. 책상이 4개면 책장도 4개, 책상이 16개면 책장도 16개. 누가 책 한 권 빌려보려면 일어나서 옆 책상까지 가야 하고, 그 사이 시간이 한참 걸린다.

문제가 있다. 어느 날 한 사람이 갑자기 책 1,000권이 필요해졌다. 자기 책장은 100권밖에 못 꽂는다. 옆 책상에 가서 책을 빌려와야 하는데, 옆 책상의 책장은 작아서 그것도 부족하다. 그렇다고 모두에게 1,000권 책장을 사줄 수도 없다 — 평소엔 100권만 쓰는데 2~3년에 한 번 1,000권 쓸 일이 있을 뿐이다. 너무 낭비다.

해결책? **사무실 한가운데에 큰 도서관을 만든다.** 모든 사람이 자기 책상에서 도서관까지 짧은 통로로 연결되어 있다. 평소엔 자기 책장에서 꺼내쓰지만, 1,000권 필요하면 도서관에서 빌려온다. 다 쓰면 반납. 다른 사람도 같은 도서관을 공유한다.

이게 CXL이다. **"각 CPU·GPU에 메모리를 따로 박는" 기존 방식 → "공유 메모리 풀(pool)을 만들어 필요한 만큼 빌려쓰는" 새 방식**.

> **용어 박스 — 핵심 4개**
> - **CXL**(Compute Express Link, 시엑스엘): "컴퓨트 익스프레스 링크". CPU·GPU·메모리·가속기를 묶는 표준 도로. 2019년 인텔 주도로 발표.
> - **PCIe**(피씨아이 익스프레스): 컴퓨터 내부의 기존 고속도로. CXL은 PCIe 5.0/6.0의 물리층 위에 깔린다. 즉, 도로 자체는 PCIe인데 그 위를 달리는 차의 규칙(프로토콜)이 다른 것.
> - **메모리 풀링**(Memory Pooling): 여러 CPU/GPU가 한 메모리 통(풀)에서 필요한 만큼 빌려쓰는 구조. 도서관 비유의 본질.
> - **캐시 일관성**(Cache Coherency): 여러 컴퓨터가 같은 데이터를 보고 있을 때 "한쪽이 바꾸면 다른 쪽도 즉시 안다"는 약속. 도서관 책에 누가 메모를 적으면 전 사용자가 같은 메모를 보는 것과 같음.

### 1-2. 좀 더 들어가기 — "왜 PCIe만으로는 안 됐나"

기존 PCIe도 CPU와 외부 카드(GPU·SSD)를 연결한다. 그런데 PCIe에는 치명적 약점이 있다. **"캐시 일관성이 없다"**.

쉬운 비유. 회사에서 김 대리와 박 과장이 같은 엑셀 파일을 쓴다고 하자. 김 대리가 자기 PC에 파일을 복사해서 작업한다. 박 과장도 자기 PC에 복사. 두 사람이 동시에 다른 줄을 수정하면? 결과는 어떻게 합치나? 누군가 한 명이 수동으로 합쳐야 한다. 실수하면 데이터가 깨진다.

PCIe로 GPU에 데이터를 보내면 정확히 이 일이 벌어진다. CPU 메모리에 있던 데이터를 GPU 메모리로 "복사"한다. 두 곳이 따로 살림 차린다. 동기화는 프로그래머가 직접 코드로 해줘야 한다. 코드 한 줄만 빠지면 결과가 엉망.

CXL은 이걸 자동화한다. **CPU 메모리와 GPU 메모리를 "한 살림"으로 본다.** CPU가 한 칸을 바꾸면 GPU도 그 변경을 즉시 안다 (캐시 일관성). 프로그래머가 동기화 코드를 짤 필요가 없다. 마치 구글 시트에 두 사람이 동시에 입력하면 실시간으로 보이는 것과 같다.

### 1-3. 더 깊이 — CXL이 가져온 3대 혁명

| 혁명 | Before(PCIe만) | After(CXL) | 비유 |
|---|---|---|---|
| **메모리 확장** | CPU 슬롯에 D램 꽂는 게 한계. 12개 슬롯이면 끝. | CXL 슬롯으로 외부 메모리 무한 확장 가능 | "옷장 12개"에서 "창고 통째로" |
| **풀링·공유** | 각 CPU 자기 D램만 사용. 남아도 못 빌려줌 | 여러 CPU가 한 풀에서 빌려씀 | "가구당 식자재"에서 "동네 공동냉장고" |
| **이종 가속기 통합** | GPU·FPGA·AI 칩 따로 메모리 관리 | CPU·GPU·AI칩 모두 한 메모리 공유 | "각자 도시락"에서 "뷔페" |

### 1-4. 한 문장 정의

> **CXL은 PCIe 5.0/6.0 위에 얹혀, CPU·GPU·메모리·가속기 간에 "캐시 일관성" 있는 통신과 "메모리 풀링"을 가능하게 하는 차세대 인터커넥트 표준이다.**

## 2. CXL 표준 진화 — 1.0부터 3.2까지 한눈에

### 2-1. 연도별 마일스톤

| 버전 | 발표 연도 | 핵심 추가 기능 | 한 줄 의미 |
|---|---|---|---|
| **CXL 1.0** | 2019.3 | 기본 프로토콜(io·cache·mem) 정의 | "도로 설계도 발표" |
| **CXL 1.1** | 2020.6 | 컴플라이언스(인증) 강화 | "도로 시공 검사 기준" |
| **CXL 2.0** | 2020.11 | **메모리 풀링·스위칭·핫플러그** | "여러 차선 + 신호등 추가" — 진짜 풀링 가능 |
| **CXL 3.0** | 2022.8 | **PCIe 6.0 기반(64 GT/s), 멀티-레벨 스위칭, P2P** | "고가도로 + 인터체인지" — GPU끼리 직접 통신 |
| **CXL 3.1** | 2023.11 | TSP(Trusted Security Protocol), GIM(Global Integrated Memory) | "도로 보안 카메라 + 통합 GPS" |
| **CXL 3.2** | 2024.12 | FM-API(Fabric Manager), 모듈 진단, HBM 호환 강화 | "교통 관제 시스템 + HBM 도로 연결" |

> **용어 박스**
> - **GT/s**(기가 트랜스퍼/초): 1초에 데이터를 몇 십억 번 전송하는지. PCIe 5.0은 32 GT/s, 6.0은 64 GT/s. 두 배 빨라짐.
> - **P2P**(Peer-to-Peer): GPU↔GPU가 CPU를 거치지 않고 직접 통신. NVLink와 같은 개념.
> - **Fabric**(패브릭): 여러 디바이스를 그물망처럼 연결한 구조. 도시 도로망 같은 것.

### 2-2. 3대 프로토콜 — io·cache·mem 분해

CXL은 한 케이블 위에 3개 채널을 동시에 흘린다. 마치 한 도로에 자동차·자전거·보행자 차선을 다 그려놓은 것과 같다.

| 프로토콜 | 무엇을 하나? | 비유 | 누가 쓰나 |
|---|---|---|---|
| **CXL.io** | PCIe 호환 통신 (디바이스 발견, 설정, DMA) | "기존 자동차 도로" | 모든 디바이스 (필수) |
| **CXL.cache** | 디바이스(가속기)가 호스트(CPU) 메모리에 캐시 일관성 접근 | "프린터가 PC 메모리를 직접 읽음" | 가속기, SmartNIC |
| **CXL.mem** | 호스트(CPU)가 디바이스 메모리(외부 D램 모듈)에 직접 접근 | "PC가 외장하드를 메모리처럼 씀" | 메모리 확장 모듈 |

쉬운 비유로:
- **CXL.io**는 "기본 도로 — 누가 누구인지 인사하고 짐 옮기는 길"
- **CXL.cache**는 "가속기가 CPU 노트를 같이 보는 길"
- **CXL.mem**은 "CPU가 외부 메모리를 자기 메모리처럼 쓰는 길"

### 2-3. 버전별 깊은 차이 — 풀링이 진짜 가능해진 시점

**CXL 1.0/1.1 (2019~2020)**: 한 호스트(CPU)와 한 디바이스(메모리/가속기)의 1:1 연결만 됐다. 메모리 확장은 가능했지만 "풀링"은 불가. 비유하면 "한 사람당 한 도서관" — 공유는 안 됨.

**CXL 2.0 (2020)**: 여기서 진짜 혁명이 시작됐다. **스위치(Switch)**가 추가되어 여러 호스트가 한 메모리 풀을 나눠 쓸 수 있게 됐다. 또 **핫플러그(Hot-Plug)** — 서버 끄지 않고 메모리 모듈을 꽂고 빼는 기능. 운영 중인 데이터센터에 "오늘 메모리 한 쪽 더 추가"가 가능해졌다.

**CXL 3.0 (2022)**: PCIe 6.0 기반으로 속도 2배. **멀티-레벨 스위칭** — 스위치 아래 또 스위치를 둘 수 있어 수백~수천 디바이스 연결 가능. **P2P** — GPU끼리 CPU 안 거치고 직접. 진짜 "데이터센터 패브릭"이 된 시점.

**CXL 3.1 (2023)**: 보안 추가. **TSP**로 메모리 풀에 접근하는 사용자를 암호화·인증. 멀티 테넌트(여러 고객) 클라우드에서 필수.

**CXL 3.2 (2024.12)**: **FM-API**(Fabric Manager API) — 풀 전체를 통합 관리하는 표준 API. 마치 "건물 전체 전기·수도를 한 화면에서 보는 BMS". 또 HBM(고대역폭 메모리)과의 호환 강화로 "GPU 옆 HBM이 부족할 때 CXL 풀에서 보충"이 본격화.

### 2-4. 다음 단계 — CXL 4.0 예고 (2026~2027 예상)

CXL 컨소시엄은 2026~2027년 PCIe 7.0(128 GT/s) 기반의 CXL 4.0을 예고했다. 속도 다시 2배, 광 인터커넥트(Optical Interconnect) 통합으로 데이터센터 단위가 아닌 **건물·캠퍼스 단위 메모리 풀** 구상. AI 시대 "한 모델이 100TB 메모리 필요" 시나리오에 대비.

## 3. CXL 디바이스 3종 — Type 1·2·3 완전 분해

CXL 표준은 디바이스를 3종류로 나눈다. **Type 1, Type 2, Type 3**. 헷갈리기 쉬우니 표·비유로 깔끔히.

### 3-1. 한눈에 비교

| 구분 | Type 1 | Type 2 | Type 3 |
|---|---|---|---|
| **무엇?** | 메모리 없는 가속기 | 메모리 있는 가속기 | 순수 메모리 확장 모듈 |
| **사용 프로토콜** | io + cache | io + cache + mem | io + mem |
| **대표 제품** | SmartNIC(스마트 네트워크 카드), FPGA | GPU, AI 가속기, FPGA(메모리 탑재) | DDR5 모듈을 CXL로 연결한 메모리 확장 카드 |
| **비유** | "프린터가 PC 메모리 직접 읽음" | "GPU가 자기 메모리 + CPU 메모리 둘 다 봄" | "외장 메모리, CPU의 추가 D램" |
| **시장 규모(2025E)** | 작음 (~$0.1B) | 중간 (~$0.3B, GPU 통합 시) | 가장 큼 (~$1B+) |
| **누가 만드나?** | Mellanox(엔비디아), Intel, Pensando(AMD) | NVIDIA, AMD, Intel, AI 가속기 스타트업 | 삼성, SK하이닉스, Micron, Astera Labs(컨트롤러) |

### 3-2. Type 1 — 가속기·SmartNIC

**무엇?** 자체 메모리는 없고, CPU 메모리를 캐시 일관성 있게 가져다 쓰는 디바이스.

**예시**: SmartNIC(스마트 네트워크 인터페이스 카드). 옛날 NIC은 단순히 데이터를 받아 CPU에 넘겼다. 이제 SmartNIC은 CPU 일을 일부 가져와 직접 처리한다 (네트워크 보안, 가상화, 압축 등). 그러려면 CPU의 데이터에 자유롭게 접근해야 한다 → CXL.cache가 필요.

**비유**: 회사에 들어온 우편물을 처리하는 비서가 있다. 옛날에는 비서가 "우편 왔습니다" 알리고 사장이 직접 봤다(NIC). 이제는 비서가 사장 책상의 메모장(메모리)을 같이 보면서 알아서 분류해서 전달한다(SmartNIC + CXL.cache).

**대표 제품**: NVIDIA BlueField, AMD Pensando, Intel IPU.

### 3-3. Type 2 — GPU·AI 가속기

**무엇?** 자체 메모리(보통 HBM)가 있고, 동시에 CPU 메모리도 캐시 일관성 있게 접근하는 디바이스.

**예시**: NVIDIA H100·B100, AMD MI300, Google TPU 등 AI 가속기. 자기 HBM 80GB가 있지만, 모델 크기가 200GB면 부족 → CPU 메모리(D램 1TB)를 같이 봄.

**비유**: 자기 책상에 책 100권 꽂은 사람이, 옆 사람 책상의 책 1,000권도 자유롭게 빌려쓰면서 페이지 메모를 공유. 실시간 동기화.

**왜 중요한가?** **AI 모델이 점점 커진다.** GPT-4급은 수십~수백 GB, GPT-5/6은 TB급 추정. HBM은 비싸서 한 GPU에 많이 못 박는다. CXL로 외부 D램을 끌어쓰면 "HBM은 자주 쓰는 데이터, CXL D램은 가끔 쓰는 데이터"로 계층화 가능.

**대표 제품**: NVIDIA Grace Hopper(GH200) — CPU(Grace, ARM)와 GPU(Hopper)를 NVLink-C2C로 묶은 CXL-라이크 구조. NVIDIA는 자체 NVLink를 밀지만 CXL.mem 호환도 점진 추가 중.

### 3-4. Type 3 — 메모리 확장 모듈 (가장 핫함)

**무엇?** GPU·CPU 같은 연산 능력은 없고, **순수 D램만 들어 있는 카드**. 마치 외장하드처럼 CPU에 꽂아 메모리를 늘림.

**예시**:
- **삼성 CMM-D**(CXL Memory Module - DRAM): 96GB·128GB·256GB. 2024 양산.
- **SK하이닉스 CMM-D 2.0**: 96GB(2024), 128GB(2025), 256GB(2025E).
- **Micron CZ120**: 128GB·256GB. 2024 샘플링, 2025 양산.
- **Astera Labs Leo 컨트롤러 + 메모리 모듈** OEM 키트.

**비유**: 컴퓨터 옆에 작은 책장을 하나 더 놓는 것. 책상(CPU) 위 책장(D램 12개)이 꽉 차서, 옆에 추가 책장(CXL Type 3 모듈)을 두고 거기서도 책을 빌려옴. 차이점은 그 추가 책장을 옆 사람과 공유 가능하다는 점.

**왜 가장 핫한가?** **시장 규모가 가장 크고, 표준화가 가장 잘 됐다.** Type 1·2는 가속기마다 설계가 달라 일반화가 어렵지만, Type 3은 D램 + 컨트롤러 = 단순. 누구나 만들기 쉽고, 누구나 살 수 있다. 클라우드 회사들이 "메모리 확장이 가장 시급" → 가장 먼저 채택.

> **용어 박스 — CMM-D / CME**
> - **CMM-D**(CXL Memory Module - DRAM): 삼성·SK하이닉스가 양산하는 CXL Type 3 메모리 모듈의 정식 명칭.
> - **CME**(CXL Memory Expander): 삼성이 별도 부르는 이름. CMM-D와 같은 카테고리.

### 3-5. 폼팩터(생김새) — E3.S·EDSFF·AIC

CXL Type 3 모듈은 어떤 모양인가? 3가지 표준이 경쟁 중.

| 폼팩터 | 모양 | 장점 | 비유 |
|---|---|---|---|
| **E3.S**(EDSFF E3.S) | 짧고 두꺼운 카드 (76mm × 112mm × 7.5mm) | 핫플러그 쉽고, 데이터센터 표준화 | "USB 메모리 두꺼운 버전" |
| **E3.L**(EDSFF E3.L) | 더 긴 카드 | 메모리 더 많이 탑재 | "USB 긴 버전" |
| **AIC**(Add-In Card) | 기존 PCIe 카드 형태 | 호환성 좋음 | "그래픽카드 모양" |

대세는 E3.S/E3.L. 데이터센터 1U 서버에 16~32개 모듈을 빽빽이 꽂을 수 있어 효율 최고.

## 4. 메모리 풀링·페일오버 입문 — "주방 식자재 공유"

### 4-1. 비유부터 — 동네 식당 5곳의 식자재

5개 식당이 한 골목에 있다. 각자 매일 100kg 식재료를 사놓는다. 가끔 한 식당에 단체 손님 와서 200kg 필요한데, 그날 다른 4 식당은 50kg밖에 안 쓴다. 결과? 한 식당은 부족해서 손님 돌려보내고, 4 식당은 식재료 50kg씩 남아 버림.

해결책: **공동 냉장창고를 만든다.** 5 식당이 매일 50kg씩만 들고 있고, 나머지는 공동 창고에. 누구든 모자라면 가서 가져옴. 남으면 반납. 결과? 같은 양으로 더 많은 손님을 받고, 폐기율 80% 감소.

이게 CXL 메모리 풀링이다.

### 4-2. 데이터센터에서 무슨 일이?

전형적 데이터센터 서버: CPU 1~2개 + D램 12개 슬롯 = 1~2TB 메모리. 100대 서버 = 100~200TB.

문제: **각 서버 메모리 사용률이 평균 30~40%에 불과**. Microsoft Azure 발표(2022)에 따르면 클라우드 D램의 50% 이상이 "stranded(고립)" 상태 — CPU는 다른 서버에서 일하는데 그쪽 메모리는 남아돈다. 이걸 빌려쓸 방법이 없었다.

**CXL 메모리 풀링** = 100 서버의 D램 일부를 한 풀(pool)에 넣고, 모든 CPU가 빌려쓴다. Microsoft 자체 시험에서 **"메모리 풀링으로 9~10% 비용 절감"** 결과 발표 (2023). 클라우드 회사 한 곳당 연간 수천억 원 절감 가능.

### 4-3. 페일오버(Failover) — "한쪽 책장이 무너져도 다른 책장이 받아줌"

페일오버는 "한 부분이 고장 나면 다른 부분이 자동으로 그 일을 대신 받는" 기능.

CXL 풀에서: 한 메모리 모듈이 고장 나면? 풀 안의 다른 모듈이 데이터를 미리 복사해놨다가 즉시 인계. 사용자는 "메모리 좀 느려졌네" 정도만 느낌, 다운타임 0.

비유: 공동 도서관에 책이 항상 2부씩 있다. 한 부가 찢어지면 즉시 다른 부로 대출. 빌리는 사람은 모름.

이걸 가능케 하는 게 CXL 2.0/3.0의 **Switch + Fabric Manager**. 풀의 상태를 실시간 감시하고, 자동으로 라우팅 변경.

### 4-4. 풀링의 한계 — "거리(latency)" 문제

좋은 점만 있는 건 아니다. 공동 창고가 멀면 가져오는 데 시간이 걸린다.

| 메모리 위치 | 접근 시간(latency) | 비유 |
|---|---|---|
| CPU 자체 D램 (DDR5) | 80~100 ns | 책상 위 책 (1초) |
| CXL 1-hop (직접 연결) | 150~250 ns | 옆 책상 (3초) |
| CXL 2-hop (스위치 경유) | 300~500 ns | 옆 방 (10초) |
| HBM (GPU 옆) | 50~70 ns | 손에 쥔 책 (0.5초) |
| SSD | 50,000~100,000 ns | 창고 (수 분) |

CXL은 D램(80ns)보다 **2~5배 느리다**. "자주 쓰는 데이터"에는 부적합. 대신 "가끔 쓰지만 용량이 커야 하는 데이터"에 최적. 이 계층화를 SW로 잘 관리하는 게 핵심.

> **용어 박스 — Tiered Memory**
> - **Tiered Memory**(계층 메모리): 빠른 메모리(HBM·D램) + 느린 메모리(CXL·SSD)를 자동으로 데이터 종류에 따라 배치. OS·하이퍼바이저가 결정. 리눅스 5.18부터 일부 기능 도입.

## 5. AI 데이터센터 적용 — GPU 메모리 부족의 구원자

### 5-1. AI 시대의 핵심 병목 — "메모리 벽(Memory Wall)"

LLM(거대언어모델)을 학습·추론하려면 모델 파라미터를 메모리에 다 올려야 한다.

| 모델 | 파라미터 | 필요 메모리 (FP16) | 필요 메모리 (FP8/INT8) |
|---|---|---|---|
| GPT-3 | 175B | 350GB | 175GB |
| GPT-4 (추정) | ~1.5T | 3,000GB (3TB) | 1,500GB |
| GPT-5 (추정) | ~10T | 20,000GB (20TB) | 10,000GB |
| Llama 3 70B | 70B | 140GB | 70GB |
| Llama 4 405B | 405B | 810GB | 405GB |

NVIDIA H100 한 장 HBM = 80GB. B100 = 192GB. **한 장으로는 GPT-4도 못 올린다.** 여러 장을 묶는데, GPU 16장 묶으면 1.5TB → GPT-4 가능. 하지만 H100 16장 = 약 50만 달러. 게다가 GPU 간 통신 병목이 생긴다.

### 5-2. CXL의 역할 — "외부 메모리 풀로 HBM 보강"

CXL Type 3 모듈로 D램 풀(예: 8TB)을 만들고, 여러 GPU가 공유. **자주 쓰는 가중치는 HBM, 가끔 쓰는 가중치는 CXL D램**으로 계층화.

비유: 요리사가 자주 쓰는 양념(소금·후추)은 손 닿는 곳에, 가끔 쓰는 양념(특수 향신료)은 옆 선반에. 다 같은 곳에 두면 요리 도중 헷갈림.

**Microsoft Azure 사례**: 2024 발표 — CXL 풀로 GPU 클러스터 메모리 효율 30% 개선, 같은 GPU로 1.5배 큰 모델 학습 가능. AWS도 2025 Trainium 차세대에 CXL 도입 예정.

### 5-3. KV Cache 오프로드 — LLM 추론의 새로운 수요

**KV Cache**(키-값 캐시)란? LLM이 답을 생성할 때, 앞 단어들의 정보를 모두 메모리에 저장해두는 공간. 답이 길수록 KV Cache가 커진다.

문제: GPT-4급에서 답 한 번에 KV Cache가 수십 GB. HBM 80GB 중 절반을 KV Cache가 점유 → 모델 자체에 쓸 메모리 부족.

해결: **KV Cache를 CXL D램 풀로 오프로드**. 자주 참조 안 되는 과거 토큰의 KV는 CXL로 보내고, 최신 토큰만 HBM. 추론 처리량 1.5~2배 향상 사례 보고 (Meta, 2024).

이게 **CXL의 가장 큰 AI 시대 기회**. 추론 데이터센터 모든 곳이 KV Cache 오프로드 수요 발생.

### 5-4. 분산 학습·체크포인트

대형 모델 학습 시 GPU 수천 장이 협력. 중간에 한 장이 고장 나면? 처음부터 다시? — 손실 큼. 그래서 주기적으로 **체크포인트**(중간 저장)를 한다.

체크포인트를 SSD에 저장하면 너무 느려 학습 시간 길어짐. CXL D램 풀에 저장하면 SSD 대비 100배 빠름. **체크포인트 시간 5분 → 5초** 사례.

### 5-5. CXL이 GPU 시장을 위협하는가?

NVIDIA는 자체 NVLink·NVSwitch로 GPU 간 통신을 묶는다. CXL이 직접 경쟁자다.

NVIDIA의 대응: **NVLink Fusion**(2025 발표) — 자체 NVLink를 외부에 개방하고, CXL과 호환 모드 추가. 즉 "내 길을 쓰면서 CXL 표준도 들어와"라는 양다리.

결론: NVIDIA GPU는 자체 NVLink + CXL 호환 모드 둘 다. CPU 진영(Intel·AMD)은 CXL 풀 베팅. **AMD MI300X**는 적극적 CXL 채택 — Infinity Fabric을 CXL로 확장. 2026년 이후 본격 충돌.

4. **풀라인업**: 컨트롤러뿐 아니라 리타이머·스위치·케이블까지 — "원스톱 솔루션".

**재무**:
- 2023 매출 $116M → 2024 $396M(+241%) → 2025E $1.2B(+200%)
- GPM 75~78% (Fabless 반도체 최상위 수준)
- 영업이익률 30%+ 진입 중
- 무차입 + 현금 보유 $700M (IPO 대금)

**리스크**:
1. **밸류에이션 너무 비쌈** — EV/Sales 25~30x (S&P 500 평균 2.5x). 약간 실망스러운 분기 실적이면 -30% 가능.
2. **고객 집중** — 매출의 60%+가 NVIDIA AI 서버에서. NVIDIA가 자체 칩 내재화하면 직접 타격.
3. **경쟁 진입** — Marvell(MRVL)이 Tanzanite 인수로 추격 중. Microchip·Rambus도 점유 노림.

**Bull/Base/Bear (12개월 시각)**:
- **Bull**: AI 서버 폭발 + 클라우드 빅4 풀가속. 매출 $2B+, 주가 +50%.
- **Base**: 매출 $1.5B, 멀티플 약간 축소. 주가 +10~15%.
- **Bear**: NVIDIA 내재화 또는 분기 실적 둔화. 주가 -30~40%.

### 7-2. Marvell Technology (MRVL) — Tanzanite 인수로 2nd 컨트롤러

**한 줄**: "Astera에 가장 위협적인 추격자". 데이터센터 인프라 칩 풀라인업 + Tanzanite 인수로 CXL 컨트롤러 진입.

**기본 정보**:
- 본사: 미국 캘리포니아 산타클라라
- 시가총액: 약 $70B
- 매출(2025E): 약 $7B
- 매출 성장률: YoY +35% (AI·데이터센터 부문)

**CXL 진입 경위**:
- 2022.5: **Tanzanite Silicon Solutions** 인수. CXL 컨트롤러 전문 스타트업.
- 2024: 첫 CXL 2.0 컨트롤러 양산.
- 2025: CXL 3.0 컨트롤러 샘플링.

**강점**:
1. **풀라인업 시너지** — 옵티컬 DSP, 이더넷 스위치, ARM CPU(2024 인수), 커스텀 ASIC까지. "데이터센터 한 박스 채울 수 있는 칩 다 만든다".
2. **AI 커스텀 ASIC 수혜** — Amazon Trainium·Google TPU 일부 칩이 Marvell 설계. CXL이 같이 들어감.
3. **고객 다변화** — Astera는 클라우드 빅4 집중, Marvell은 통신·엔터프라이즈도 광범위.

**약점**:
1. **CXL 후발 주자** — Astera 대비 1~1.5년 늦음. 점유율 한계.
2. **광범위한 사업 → 집중도 부족** — 투자자에게 "Marvell의 CXL 노출"이 명확히 안 보임.

### 7-3. Microchip Technology (MCHP) — CXL 메모리 컨트롤러 다크호스

**한 줄**: "조용한 강자". 임베디드 MCU 강자였으나 SerDes·CXL 컨트롤러 영역에서 의외의 점유.

**기본 정보**:
- 본사: 미국 애리조나 챈들러
- 시가총액: 약 $35B
- 매출(2025E): 약 $4.5B (사이클 회복 중)

**CXL 제품**:
- **SMC-2000 시리즈**: CXL 2.0 메모리 컨트롤러. 2023 출시, 2024 양산.
- 데이터센터 D램 모듈에 OEM 공급. 삼성·SK하이닉스·Micron 일부 모듈에 탑재.

**강점**:
1. **검증된 ASIC 설계 능력** — 30년 임베디드 노하우.
2. **자동차·산업 시장과 시너지** — 자동차 전동화·자율주행도 CXL 채택 검토 중.

**약점**:
- 매출에서 CXL 비중 작음 (2025E ~3%). 순수 베팅용으론 부적합.

### 7-4. Montage Technology (688008.SH) — 중국 CXL 1위

**한 줄**: "중국판 Astera". RDIMM 칩셋(서버 D램 컨트롤러) 글로벌 1위 → CXL로 자연 확장.

**기본 정보**:
- 본사: 중국 상하이
- 상장: 상해 STAR Market
- 시가총액: 약 ¥80B (~$11B)
- 매출(2025E): 약 ¥4B (~$0.55B)

**RDIMM 칩셋이 뭔가?**
서버용 D램 모듈(RDIMM)에 들어가는 작은 컨트롤러 칩. JEDEC 표준 인증을 받은 회사만 만들 수 있다. 글로벌에 **Montage·Renesas·Rambus 단 3사**.

**왜 CXL로 자연 확장?**
RDIMM 칩셋과 CXL 메모리 컨트롤러는 기술 기반이 비슷. 메모리 신호 처리 노하우 공통. Montage는 2022년부터 CXL MXC(Memory eXpansion Controller) 양산.

**강점**:
1. **중국 자국 수요 독점** — 미·중 갈등으로 Astera 칩 수입 어려워지면 Montage가 자동 1위.
2. **JEDEC 표준 인증** — 진입 장벽 매우 높음.
3. **고성장**: RDIMM 사이클 회복 + CXL 추가로 매출 YoY +60%+.

**리스크**:
1. **미국 제재 가능성** — Astera 대체재로 부각될수록 제재 리스크.
2. **중국 외 진출 한계** — 글로벌 클라우드는 거의 못 침투.

### 7-5. Rambus (RMBS) — IP·컨트롤러 라이센서

**한 줄**: "CXL의 ARM". 자체 칩 양산은 적고, IP를 설계한 후 라이센스로 매출.

**기본 정보**:
- 본사: 미국 캘리포니아 산호세
- 시가총액: 약 $7B
- 매출(2025E): 약 $0.6B

**제품**:
- **CXL 1.1/2.0/3.0 컨트롤러 IP** — 다른 회사가 자기 칩에 통합해 사용.
- **CXL 메모리 인터페이스 IP** — D램 컨트롤러 IP 글로벌 1위 (JEDEC 표준에 강함).
- **CXL Memory Initiative** — 2022 출범, 컨트롤러·인증 칩 양산.

**강점**:
1. **IP 라이센스 매출 안정** — 한 번 설계해 여러 고객에 팜. GPM 65%+.
2. **JEDEC 영향력** — D램 표준 제정에 깊이 관여. CXL 표준에도 핵심 기여.

**약점**:
- 자체 칩 매출은 작아 "성장 베팅"으론 다소 약함.

### 7-6. SK하이닉스 — 96GB·128GB CMM-D 양산, CXL 2.0/3.0

**한 줄**: "한국 메모리 1위 + HBM 1위 + CXL 모듈 1위 후보". CXL의 한국 대표주.

**기본 정보** (2026-05-11 기준):
- 본사: 한국 이천
- 시가총액: 약 ₩170조 (~$120B, 환율 1,420원 가정)
- 매출(2025E): 약 ₩90조

**CXL 제품**:

- **CMM-D 128GB**: 2025년 양산 시작.
- **CMM-D 256GB**: 2025 하반기 ~ 2026 초 예상.
- **CMM-DDR5 풀링 솔루션**: 여러 모듈을 한 풀로 묶는 통합 키트. Samsung과 경쟁 중.

**강점**:
1. **HBM과 결합 시너지** — HBM(GPU 옆) + CXL D램(외부 풀)을 한 솔루션으로 제안. NVIDIA·AMD가 환영.
2. **클라우드 빅4 채택** — Microsoft·Meta가 SK하이닉스 CMM-D 채택 발표.
3. **D램 원가 경쟁력** — 1b nm 공정으로 양산, Micron보다 원가 우위.

**약점**:
- **CXL 매출 비중 작음** (2025E ~1%). 단기 주가 영향 제한.
- D램 사이클 의존성. CXL이 좋아도 D램 가격이 빠지면 주가 하락.

**투자 포인트**:
- HBM·CXL·낸드 3대 축으로 2027년 매출 ₩140조+ 가능. CXL은 2027 이후 매출의 5~8% 차지 예상.

### 7-7. 삼성전자 — CMM-D 모듈 + CME

**한 줄**: "메모리 종합 1위. CXL에선 SK하이닉스에 약간 뒤처졌으나 빠르게 추격".

## 9. 밸류에이션 — 어떻게 평가하나 + 현재 위치

### 9-1. CXL 섹터를 어떻게 평가하나?

CXL 관련 종목은 크게 3가지로 나뉘고, 각각 평가 멀티플이 다르다.

| 종목 유형 | 주 멀티플 | 왜? | 비유 |
|---|---|---|---|
| **컨트롤러 Fabless**(ALAB, MRVL) | EV/Sales, PEG, FCF Yield | 매출 성장률 100%+, 이익은 작음. 성장주 베팅. | "성장 가속하는 스타트업" |

> **용어 박스**
> - **EV/Sales**(이브이 슬래시 세일즈): 기업가치 ÷ 매출. 이익 안 나는 성장주에 사용. 낮을수록 저평가.
> - **PEG**(펙): P/E ÷ 성장률. 1.0이 적정. AI 성장주는 1.5~2.0 허용.
> - **P/B**(피비): 시가총액 ÷ 자기자본. 사이클 메모리는 0.8~2.0 사이 밴드.
> - **ROE**: 자기자본 수익률. P/B와 짝꿍. ROE 높으면 P/B 높아도 정당.
> - **Replacement Cost**(대체원가): "지금 이 공장을 다시 짓는 데 드는 돈". 메모리 사이클 바닥에서 시가총액이 대체원가보다 낮으면 매수 시그널.

## 12. 신기술·차세대 — CXL Beyond 2027

### 12-1. CXL + Optical Interconnect (광 인터커넥트)

전기 신호는 거리 1m+에서 손실 큼. **광(빛) 신호로 바꾸면 100m+ 무손실**. 데이터센터 한 건물 전체를 CXL 풀로 묶을 수 있음.

- **Lightmatter** (스타트업): 광 인터커넥트 + CXL 통합 칩 개발.
- **Ayar Labs** (스타트업, NVIDIA·Intel 투자): 광 패키징 통합.
- **2027~2028 상용화 전망**.

### 12-2. CXL + CPO (Co-Packaged Optics)

CPO = 광학 송수신기를 칩 패키지 안에 통합. CXL 신호를 칩 옆에서 바로 광으로 변환.

- **Broadcom·Marvell·Astera Labs**가 모두 CPO + CXL 통합 솔루션 개발 중.

### 12-3. CXL Memory Sharing for AI Training

차세대 AI 학습에서 **수천 GPU가 모델 가중치를 공유 메모리로 둠**. NVLink·UALink·CXL이 경쟁.

- **CXL 3.0 P2P**가 핵심: GPU↔GPU 직접 통신, CPU 안 거침.
- **Meta·OpenAI·xAI**가 차세대 학습 인프라에 CXL 3.0 P2P 검토.

### 12-4. CXL + DPU/IPU 통합

DPU(Data Processing Unit) = SmartNIC + CPU + 가속기 통합 칩. CXL.cache로 호스트 메모리 직접 접근.

- **NVIDIA BlueField 4** (2026E): CXL 2.0 통합.
- **AMD Pensando 2nd gen**: CXL 통합.

## 13. CCIX·OpenCAPI·Gen-Z — 죽은 경쟁 표준들

CXL 이전에 비슷한 인터커넥트 표준이 여러 개 있었다. 다 사라졌다. 왜?

| 표준 | 주도 | 결과 | 사망 원인 |
|---|---|---|---|
| **CCIX**(Cache Coherent Interconnect for Accelerators) | ARM, AMD, Xilinx | 2020 사실상 종료, CXL에 흡수 | 산업 분열, 채택 부진 |
| **OpenCAPI** | IBM | 2022 종료, CXL 컨소시엄에 통합 | IBM Power 의존 |
| **Gen-Z** | HPE, Dell | 2021 종료, CXL에 자산 이전 | 표준 너무 야심차서 늦음 |
| **NVLink** | NVIDIA | 자체 노선 유지 | NVIDIA GPU에 한정 |

**교훈**: 인터커넥트 표준은 "한 회사 단독 표준은 못 살아남는다, 컨소시엄 표준이 이긴다". CXL이 살아남은 이유: Intel·AMD·NVIDIA·삼성·SK·MS·Google·AWS 등 90+ 회원사 참여.

> **용어 박스 — CCIX**
> - **CCIX**(시식스 또는 씨씨아이엑스): Cache Coherent Interconnect for Accelerators. ARM 진영의 캐시 일관성 인터커넥트 표준. 2017~2020 활동, CXL 등장 후 사실상 흡수.

## 14. UALink — 새로운 경쟁자 (2024~)

2024년 5월, **UALink**(Ultra Accelerator Link) 컨소시엄 발족. 멤버: AMD, Intel, Broadcom, Cisco, Google, HPE, Meta, Microsoft.

**무엇?** 차세대 AI 가속기(GPU·TPU·AI 칩) 간 200 Gbps+ 초고속 연결 표준. NVLink에 대항.

**CXL과의 관계**?
- **보완**: UALink는 가속기↔가속기 (NVLink 자리), CXL은 가속기↔메모리·CPU. 영역 다름.
- **경쟁 가능성**: UALink가 메모리 풀링도 흡수하면 CXL의 일부 영역 침범.
- **현재 견해**: 두 표준 공존. NVIDIA만 NVLink 단독.

CXL은 **AI 시대 데이터센터의 새 동맥**이다. HBM이 GPU 옆 단거리 고속도로라면, CXL은 도시 전체를 잇는 순환도로. 두 길은 경쟁이 아니라 **보완**이다. AI 모델이 커질수록 HBM과 CXL 둘 다 더 많이 필요하다.

시장은 2024 약 5억 달러에서 2030년 150~200억 달러로 30~40배 성장 예상. CAGR 60%+의 폭발적 성장. 단, 2026~2027이 채택 변곡점이며, CPU 세대 교체(Intel Granite Rapids·AMD Turin) 속도와 클라우드 빅4의 도입 가속 여부가 핵심 변수.

### 17-2. Top Pick 3

#### 1위: Astera Labs (ALAB) — 순수 베팅의 정점

- **왜?**: CXL 컨트롤러 글로벌 1위. AI 서버 인터커넥트의 길목 독점. 매출 성장률 +150~200%.
- **목표가**: $90~130 (12개월).
- **리스크**: EV/Sales 25x 부담. 분기 실적 미스 시 -30% 즉발.
- **포지션**: 포트폴리오의 5~7%. 분할 매수 권장.

#### 2위: SK하이닉스 (000660) — 한국 메모리 + CXL 통합

- **왜?**: HBM 1위 + CMM-D 1위 후보. CXL 매출 비중은 작지만 D램+HBM+CXL 3중 동력.
- **목표가**: ₩350,000~495,000 (12~24개월).
- **리스크**: D램 사이클 하락. 2027~2028 둔화 시 -30%.
- **포지션**: 포트폴리오의 8~12%.

#### 3위: Marvell (MRVL) — 분산 베팅

- **왜?**: 컨트롤러 2위 + 옵티컬·이더넷·ARM CPU 풀라인업. AI ASIC(Trainium·TPU 일부) 동시 수혜.
- **목표가**: $80~115 (12개월).
- **리스크**: ALAB 대비 CXL 노출 희석. 매출 다변화로 임팩트 작음.
- **포지션**: 포트폴리오의 3~5%.

### 17-4. 마지막 한 마디

CXL은 "조용한 혁명"이다. NVIDIA H100/B100만큼 시끄럽진 않지만, 그 GPU들이 "메모리 부족"으로 발목 잡히는 현실을 해결하는 핵심 기술. **AI가 더 커질수록 CXL은 더 필요해진다**. 투자 타이밍은 지금~2027이며, 컨트롤러(ALAB)에 집중하되 메모리 모듈(SK하이닉스)로 분산 권장.
