## 0. 본 리포트의 위치 — 왜 PIM만 따로 보나

같은 폴더에 발주된 자매 리포트 `차세대메모리_종합분석.md`는 차세대 메모리(MRAM·ReRAM·PCM·FeRAM·STT-MRAM 등) 전체를 한 호흡에 다룬다. 그러나 **PIM(Processing-In-Memory)**만큼은 다음 이유로 단독 리포트가 필요하다.

1. **PIM은 메모리이자 동시에 연산기**다. 다른 차세대 메모리(MRAM·ReRAM)가 "더 빠르고 안 꺼지는 저장소"라면, PIM은 "**저장소가 직접 계산까지 한다**"는 패러다임 변화다.
2. PIM 시장은 2024년 약 **3억 달러(0.3조 원)**에서 2030년 **200억 달러(28조 원)+**로 67배 성장이 예상되는 구간에 있다(Yole Group, McKinsey).
3. AI 추론(Inference) 시대로 넘어가면서 **GPU만으로는 전력·메모리 대역폭 한계**가 분명해졌고, NVIDIA·AMD조차 차세대 GPU에 PIM 일부 통합을 검토 중이다.
4. **삼성전자 HBM-PIM(Aquabolt-XL)**, **SK하이닉스 AiM**은 이미 양산·샘플 단계로 진입했고, 2026~2028년 상용화 본격화가 예상된다.

> 💡 **본 리포트가 새로 더하는 것**
> 1. PIM 5종(NMC·PNM·PIM·IMC·CIM) 분류와 한국 투자자 관점 정리
> 2. 메모리 종류별 PIM(DRAM·HBM·LPDDR·SRAM·ReRAM·MRAM) 비교표
> 3. 글로벌 PIM 12개사 종목 심층(상장·비상장 모두)
> 4. PIM vs GPU 효율 매트릭스(전력당 추론 처리량)
> 5. 삼성·SK 본업과 PIM 옵션 가치 분리 SOTP

## 2. PIM 5분 입문 — "메모리에서 직접 계산"

### 2-1. 비유로 시작 — "창고에서 바로 요리"

#### 기존 컴퓨터(폰노이만 구조)
당신이 식당 셰프라고 하자.
- **식자재 창고(메모리, D램)** 와 **부엌(CPU/GPU)**이 200m 떨어져 있다.
- 양파 1개 필요 → 창고까지 200m 걸어가서 양파 가져옴 → 부엌 와서 썰기
- 마늘 필요 → 또 200m 가서 가져옴
- 결과: **요리하는 시간보다 걸어다니는 시간이 더 길다.**

#### PIM 구조
- **창고 안에 작은 부엌을 만든다.**
- 창고 직원이 양파·마늘을 가져오면서 **그 자리에서 바로 다진다.**
- 부엌(CPU)은 다져진 재료만 받아서 최종 조리만 한다.
- 결과: **걸어다니는 거리(데이터 이동)가 90% 줄어든다.**

이게 PIM의 본질이다. **메모리(창고) 안에 작은 연산기(부엌)를 심어서, 데이터가 멀리 이동하지 않고 그 자리에서 계산하게 만든 것**.

### 2-2. 좀 더 정확한 정의

> **PIM(Processing-In-Memory)** = 메모리 칩 내부 또는 바로 옆에 연산 로직(ALU, MAC 등)을 통합하여, CPU/GPU로 데이터를 보내지 않고 메모리 자체에서 데이터 처리를 수행하는 컴퓨팅 아키텍처.

#### 약어 풀이 박스

| 약어 | 풀이 | 한 줄 설명 |
|---|---|---|
| **PIM** | Processing-In-Memory | 메모리 안에서 계산. 가장 넓은 의미의 우산 용어 |
| **NMC** | Near-Memory Computing | 메모리 옆(같은 패키지) 연산. PIM의 약한 형태 |
| **PNM** | Processing-Near-Memory | NMC와 거의 동의어, 삼성·SK 자료에서 자주 사용 |
| **IMC** | In-Memory Computing | 메모리 셀 그 자체에서 계산. 가장 강한 형태 |
| **CIM** | Compute-In-Memory | IMC와 거의 동의어, 학계·논문에서 사용 |
| **HBM-PIM** | High Bandwidth Memory-PIM | HBM(적층 D램) + PIM. 삼성 Aquabolt-XL이 대표 |
| **AiM** | Accelerator-in-Memory | SK하이닉스의 PIM 브랜드명 |
| **AXDIMM** | Acceleration DIMM | 삼성의 서버용 DIMM 형태 PIM |
| **LPDDR-PIM** | Low Power DDR-PIM | 모바일·엣지용 저전력 PIM |
| **APU** | Associative Processing Unit | GSI Technology의 SRAM 기반 PIM 칩 |
| **MAC** | Multiply-Accumulate | "곱하고 더하기" — AI 추론의 99% 연산 |
| **TOPS** | Tera Operations Per Second | 초당 1조 회 연산 |
| **TOPS/W** | TOPS per Watt | 1와트당 처리량 = AI 칩 효율 지표 |

### 2-3. 왜 지금 갑자기 PIM이 뜨나?

3가지 트리거가 동시에 왔다.

1. **AI 추론(Inference) 폭발** — ChatGPT·Gemini·Claude 같은 LLM이 매일 수십억 번 호출됨. 학습은 1번이지만 추론은 영원히 반복되므로 **추론 비용 = AI 산업의 진짜 비용**이 됐다.
2. **메모리 벽(Memory Wall)** — CPU/GPU는 매년 빨라지는데 메모리 대역폭은 그만큼 못 따라온다. NVIDIA H100조차 LLM 추론 시 **연산 유닛의 70%가 메모리 기다리며 놀고 있다**.
3. **전력 한계** — 데이터센터 전력이 미국 전력 소비의 4%(2024) → 2030년 9%로 폭증 예상. **전력당 효율 = AI 인프라의 진짜 KPI**가 됐고, PIM이 여기서 GPU 대비 5~10배 우위.

> 💡 **숫자 환산**: NVIDIA H100 GPU 1대는 전력을 700W 사용한다. 형광등 14개. 데이터센터 1만 대 = 7MW = 작은 도시 1만 가구 전기.
> AI 추론 1번 = 구글 검색 10번 전력. ChatGPT 매일 25억 회 호출 = 구글 검색 250억 회 전력 소모.

## 3. 폰노이만 병목 입문 — 90% 시간·전력이 어디로 사라지나

### 3-1. 폰노이만 구조란?

1945년 **존 폰노이만(John von Neumann)**이 제안한 컴퓨터 구조. 80년 동안 모든 컴퓨터가 이 구조를 따른다.

```
[CPU/GPU] ←→ [버스(데이터 통로)] ←→ [메모리(D램)]
       ↑                            ↑
      계산                          저장
```

핵심 원리: **계산하는 곳(CPU)과 저장하는 곳(메모리)이 분리**되어 있다. 데이터가 필요할 때마다 메모리 → CPU → 메모리로 왕복.

### 3-2. 왜 병목이 생기나? — "10cm vs 1nm"

CPU 내부 트랜지스터(연산기) 사이 거리는 **1nm(나노미터)** 수준. 빛이 0.0000000003초에 도달.
CPU와 메모리 D램 사이 거리는 **10cm**. 빛이 0.3나노초에 도달 — **10억 배 멀다.**

게다가 데이터가 이동하려면:
1. CPU에서 신호 발생
2. 마더보드 회로 따라 이동(저항·노이즈)
3. 메모리 컨트롤러 통과
4. D램 셀 활성화
5. 데이터 읽기
6. 다시 역순으로 CPU 복귀

총 **수백 사이클(수 마이크로초)**. 그동안 CPU는 **놀고 있다.**

### 3-3. 숫자로 보는 병목 — 90%가 이동에 쓰인다

#### 전력 측면 (Sze et al., MIT, 2017)
| 작업 | 에너지 (45nm 공정) |
|---|---|
| 32-bit 정수 덧셈 (ALU 내부) | 0.1 pJ |
| 32-bit 곱셈 (ALU 내부) | 3.1 pJ |
| 32-bit DRAM 접근 (메모리 ↔ CPU) | **640 pJ** |

> 💡 **무슨 의미?** 1번 곱셈에 쓰는 전력의 **200배**가 메모리 한 번 가져오는 데 쓰인다. 즉 **AI 연산 전력의 99%가 데이터 이동에 사라진다.**

#### 시간 측면 (NVIDIA H100, LLM 추론 기준)
| 단계 | 비중 |
|---|---|
| 실제 연산(MAC) | **약 10%** |
| 메모리 대기(HBM 읽기/쓰기) | **약 70%** |
| 통신·동기화(여러 GPU 간) | 약 20% |

> 💡 **즉**: GPU의 비싼 연산 코어가 70%를 메모리 기다리며 놀고 있다. 1억 원짜리 H100 GPU의 70%가 휴식 중. 만약 PIM으로 메모리 자리에서 미리 계산하면, 같은 시간에 **2~3배** 결과가 나온다.

### 3-4. 폰노이만 병목 = 산업 전체의 거대한 비효율

전 세계 데이터센터가 매년 약 **400 TWh**(2024) 전력 소비. 한국 1년 전력 소비(약 530 TWh)의 75%. 이 중 **AI 추론에 쓰이는 비중이 30%(약 120 TWh)**.

만약 폰노이만 병목으로 90%가 데이터 이동에 사라진다면, 실제 유효한 계산은 **12 TWh** 정도. 나머지 **108 TWh**(원자력 발전소 12기 1년치)가 그냥 사라진다.

PIM이 이 중 50%만 회수해도 **54 TWh** 절감 — 한국 가정 전체 1년 전기 소비량과 맞먹는다.

## 4. PIM 5종 — NMC·PNM·PIM·IMC·CIM

같은 "메모리 안 계산"이라도 깊이가 다르다. 5가지를 구분하자.

### 4-1. 5종 한눈에

| 약어 | 풀네임 | 위치 | 강도 | 대표 제품 |
|---|---|---|---|---|
| **NMC** | Near-Memory Computing | 메모리 옆(같은 보드) | 약 | Intel Optane DIMM |
| **PNM** | Processing-Near-Memory | 메모리 옆(같은 패키지) | 중 | 삼성 AXDIMM |
| **PIM** | Processing-In-Memory | 메모리 칩 내부(별도 다이) | 강 | 삼성 HBM-PIM, SK AiM |
| **IMC** | In-Memory Computing | 메모리 셀 자체에서 계산 | 매우 강 | UPMEM, Mythic |
| **CIM** | Compute-In-Memory | IMC와 동의어(논문) | 매우 강 | Stanford·MIT 연구 |

### 4-2. 비유로 다시 — "부엌과 창고의 거리"

| 종류 | 부엌 위치 | 직원 수 | 데이터 이동 거리 |
|---|---|---|---|
| **기존(폰노이만)** | 옆 건물 | 1명(셰프) | 200m |
| **NMC** | 같은 건물 다른 층 | 2명 | 30m |
| **PNM** | 같은 층 옆 방 | 5명 | 5m |
| **PIM** | 창고 안 한쪽 코너 | 100명(작은 부엌 100개) | 30cm |
| **IMC/CIM** | 창고 선반 자체에 작은 칼이 달림 | 셀 수만큼 | 0cm |

깊이가 깊어질수록 **속도·전력 효율은 좋아지지만, 만들기 어렵고 범용성이 떨어진다.**

### 4-3. NMC — 가장 약한 형태, 가장 만들기 쉬움

#### 정의
**Near-Memory Computing** = 메모리 칩 옆에 별도의 가속기 칩을 두고, 같은 보드(DIMM·SSD)에 통합. 메모리 다이 자체는 안 건드림.

#### 대표 사례
- **Intel Optane DIMM** (단종) — 3D XPoint 메모리 옆에 컨트롤러 통합
- **Samsung Smart SSD** — SSD 컨트롤러에 ARM 코어 추가, SQL 쿼리 직접 처리
- **NGD Systems Newport** — SSD 안에 4코어 ARM, 데이터 필터링 수행

#### 강점·약점
- 강점: 기존 메모리 표준(DDR·NVMe) 그대로 사용 가능
- 약점: 데이터 이동 거리가 여전히 길어서 효율 개선 30% 수준

### 4-4. PNM — 같은 패키지, 다른 다이

#### 정의
**Processing-Near-Memory** = 메모리 다이와 연산 다이를 같은 패키지(반도체 한 통)에 적층 또는 인접 배치.

#### 대표 사례
- **삼성 AXDIMM**(Acceleration DIMM) — 서버용 DDR5 DIMM에 FPGA 가속기 통합
- **HBM 베이스 다이(Logic Die)** — HBM3e 맨 아래 컨트롤러 다이에 일부 연산 로직 (NVIDIA·AMD가 활용 중)

#### 강점
- 메모리 표준 호환
- 같은 패키지라서 데이터 이동 거리 1mm 이하

### 4-5. PIM — 진짜 PIM, 메모리 다이 안에 연산기

#### 정의
**Processing-In-Memory** = D램 다이 자체에 연산 유닛(PCU, Processing Unit) 통합. 한 칩 안에 셀과 계산기가 함께.

#### 대표 사례
- **삼성 HBM-PIM(Aquabolt-XL)** — HBM2 다이 안에 16-bit Floating Point 연산기, MAC 32개/뱅크
- **SK하이닉스 AiM(GDDR6-AiM)** — GDDR6 다이 안에 1GHz BF16 MAC 16개/뱅크
- **삼성 LPDDR-PIM** — 모바일용 LPDDR5에 PIM 통합 시제품

#### 강점·약점
- 강점: 효율 5~10배, 본격적 PIM
- 약점: D램 셀과 로직 회로의 공정이 달라서 양산이 어렵다(D램은 CMOS 공정, 로직은 FinFET 공정)

### 4-6. IMC/CIM — 메모리 셀 자체가 계산

#### 정의
**In-Memory Computing** = 메모리 셀의 물리 특성(저항·전류)을 이용해 계산을 수행. 셀 = 저장 + 연산.

가장 대표적: **저항을 곱셈으로 변환**.
- D램·SRAM 셀에 0.5V 인가 → 저항 R → 전류 I = V/R = 0.5/R
- 이걸 행렬 곱셈으로 활용: V × G(컨덕턴스) = I → 자연스럽게 곱셈 발생
- 여러 셀의 전류 합 = 덧셈 → MAC(곱·합) 한 번에 완성

#### 대표 사례
- **UPMEM DRAM-PIM** — D램 셀과 ARM 코어 결합 (실은 PIM에 가까움)
- **Mythic Analog Matrix Processor** — 플래시 메모리 셀의 저항을 가중치로 사용, 아날로그 곱셈
- **GSI Technology APU** — SRAM 셀 자체에서 비교·검색 연산
- **Crossbar ReRAM-CIM** — ReRAM 셀의 저항 변화로 곱셈

#### 강점·약점
- 강점: 효율 100배+ (이론상)
- 약점: 아날로그 노이즈, 정확도 떨어짐, AI 추론에는 OK이지만 일반 연산엔 불가

## 5. 메모리 종류별 PIM — 6가지 매트릭스

PIM은 어떤 메모리 위에 만드느냐에 따라 특성이 완전히 달라진다.

### 5-1. 한눈에 보기

| PIM 종류 | 베이스 메모리 | 대역폭 | 용량 | 전력 | 대표 회사 | 응용 |
|---|---|---|---|---|---|---|
| **DRAM-PIM** | DDR4/5 | 보통(40 GB/s) | 큼(GB) | 낮음 | UPMEM | 데이터베이스 가속 |
| **HBM-PIM** | HBM2/3/3e | 매우 높음(800 GB/s+) | 중간(GB) | 중간 | 삼성, SK | LLM 추론 |
| **LPDDR-PIM** | LPDDR4/5 | 낮음(20 GB/s) | 중간 | 매우 낮음 | 삼성 | 모바일·엣지 AI |
| **GDDR-PIM** | GDDR6/6X | 높음(500 GB/s) | 작음 | 높음 | SK(AiM) | 그래픽·AI 가속 |
| **SRAM-PIM** | SRAM | 매우 매우 높음 | 매우 작음(MB) | 낮음 | GSI, Untether | IoT, 검색 |
| **ReRAM-PIM** | ReRAM | 중간 | 작음 | 매우 낮음 | Crossbar, TetraMem | 엣지 AI |
| **MRAM-PIM** | MRAM/STT-MRAM | 중간 | 작음 | 매우 낮음 | Numem, 삼성 R&D | 보안·웨어러블 |

### 5-2. DRAM-PIM — 가장 범용, 가장 큰 시장

#### 핵심 원리
일반 DDR4/5 D램 다이 안에 ALU(산술논리장치) 또는 ARM 코어를 추가. 메모리에 저장된 데이터를 그 자리에서 직접 처리.

#### 대표 회사: UPMEM (프랑스, 비상장)
- 2015 설립, 파리 본사
- DDR4 DIMM 1개에 **128개 ARM 코어** 통합 (PIM-DRAM)
- 1 DIMM = 64GB 메모리 + 128 코어 = **PIM 1세대 상용화**
- 고객: 유전체 분석(genomics), 금융 백테스트, 데이터베이스
- 자금조달: 시리즈 C 4,000만 유로(2023), 누적 1억 달러+
- IPO 가능성: 2027~2028 (관측)

> 💡 **왜 중요한가**: 세계 최초 PIM 양산 회사. 한국 투자자가 직접 살 순 없지만, **이 회사 주가 흐름이 PIM 산업 전체의 정성적 지표**가 된다(IPO 시).

### 5-3. HBM-PIM — 한국이 1등, AI의 핵심

#### 핵심 원리
**HBM(High Bandwidth Memory)**은 D램 다이를 8~12장 쌓아 올려 한 패키지로 만든 고성능 메모리. 이 적층 D램 한 장 한 장에 연산기를 넣은 게 HBM-PIM.

#### 대표 제품 1: 삼성 HBM-PIM "Aquabolt-XL"
- 2021년 세계 최초 발표
- HBM2 베이스, **PCU(Programmable Computing Unit)** 32개 통합/다이
- 16-bit Floating Point MAC 지원
- 성능: AI 추론 워크로드에서 기존 HBM 대비 **2배 빠름, 전력 70% 절감**
- 고객: AMD MI100 데모(2022), Meta·MS R&D 단계
- 후속 모델: HBM3-PIM(2024 발표), HBM3e-PIM(2025 샘플), HBM4-PIM(2026~2027)

#### 대표 제품 2: SK하이닉스 AiM (GDDR6-AiM 우선, HBM-AiM 후속)
- 2022년 GDDR6-AiM 양산 시작
- BF16(Brain Float 16) MAC 16개/뱅크, 1GHz 동작
- 성능: GPT-2 추론에서 GPU 대비 **10배 효율, 전력 1/16**
- 후속: HBM-AiM 2026~2027 양산 목표

> 💡 **숫자 환산**: HBM 한 스택(8장 적층)에 PIM 통합 → 한 스택당 256개 MAC, 8 스택이면 2,048개 MAC. NVIDIA H100 GPU의 Tensor Core(528개) 대비 4배 많은 연산 유닛이 메모리 자리에 있음.

### 5-4. LPDDR-PIM — 모바일·엣지 AI

#### 핵심 원리
**LPDDR(Low Power DDR)**은 스마트폰·태블릿용 저전력 D램. 여기에 PIM을 넣으면 스마트폰 안에서 LLM 추론 가능.

#### 대표: 삼성 LPDDR-PIM
- 2023 시제품 공개
- 갤럭시 폰 차세대 모델(S26~S27 가능성) 탑재 검토
- 핵심: ChatGPT·Gemini를 클라우드 안 거치고 폰 안에서 처리(Privacy·Speed)

> 💡 **응용**: 갤럭시 S30(2027 예상)에서 인터넷 없이 GPT-4 급 LLM 동작 → 삼성 vs 애플 모바일 AI 경쟁의 결정타.

### 5-5. GDDR-PIM — 그래픽 메모리 + AI

#### 핵심 원리
**GDDR(Graphics DDR)**은 그래픽카드용 메모리. HBM보다 싸고, 일반 D램보다 빠르다. 이 위에 PIM을 올린 게 GDDR-PIM.

#### 대표: SK하이닉스 GDDR6-AiM
- 2022 양산
- 가격이 HBM-PIM 대비 1/3 → **AI 추론 가속기 시장의 보급형**
- 응용: 자체 추론 가속기(Meta MTIA, Google TPU 등)에 채택 가능성

### 5-6. SRAM-PIM — 가장 빠르지만 가장 작음

#### 핵심 원리
**SRAM(Static RAM)**은 CPU 캐시(L1·L2·L3)에 쓰는 매우 빠른 메모리. 작지만 빠름. 여기에 PIM을 넣으면 극도로 빠른 추론기.

#### 대표 회사
- **GSI Technology(GSIT)** — APU(Associative Processing Unit), SRAM 기반 병렬 검색
- **Untether AI** — In-Memory AI 가속기, SRAM 기반
- **Mythic** — 플래시 + SRAM 하이브리드(아날로그 PIM)

#### 강점·약점
- 강점: 가장 빠름, 정밀도 높음
- 약점: 용량이 매우 작음(MB 단위), 큰 LLM에는 부적합

### 5-7. ReRAM-PIM — 비휘발성 + 초저전력

#### 핵심 원리
**ReRAM(Resistive RAM)** 셀의 저항을 AI 가중치로 사용. 곱셈을 물리적으로 수행.

#### 대표 회사
- **Crossbar(미국, 비상장)** — ReRAM 전문, PIM 응용 R&D
- **TetraMem(미국, 비상장)** — ReRAM-CIM 칩
- **Weebit Nano(이스라엘)** — ReRAM IP

### 5-8. MRAM-PIM — 차세대 옵션

#### 핵심 원리
**MRAM(Magnetic RAM)** 셀의 자기 저항을 활용. 비휘발성 + 빠름 + 저전력.

#### 대표
- **Numem(미국, 비상장)** — MRAM-CIM
- **삼성 R&D** — STT-MRAM PIM 논문 발표(2023, 2025)

## 6. AI 추론·트랜스포머 가속에 왜 PIM이 필요한가

### 6-1. AI 추론의 본질 — Vector × Matrix

ChatGPT가 "안녕"이라는 답을 만들 때 내부에서 일어나는 일.

1. 사용자 입력 "안녕하세요" → 토큰 4개로 변환
2. 각 토큰 → **벡터(숫자 배열)** 768차원 또는 4,096차원
3. 트랜스포머 모델 안에서 **벡터 × 가중치 행렬(Matrix)** 수십억 번 반복
4. 결과 벡터 → 다시 단어로 디코딩 → "반갑습니다"

핵심: **벡터 × 행렬 곱셈 = AI 추론의 99% 작업.** 이걸 빨리·싸게 하면 AI 추론 비용이 떨어진다.

### 6-2. 비유 — "시장에서 장보기"

일반 GPU 추론:
- 시장(메모리)에서 양파(데이터) 1개 가져옴 → 부엌(GPU)에서 곱셈
- 마늘 가져옴 → 곱셈
- 양파 1만 개를 한 번에 사야 한다면? → 1만 번 왕복

PIM 추론:
- 시장 안에 작은 부엌 1만 개 → 양파 1만 개를 그 자리에서 동시에 곱셈
- 결과만 묶어서 한 번에 가져옴

**LLM 추론은 매번 수십억 개 가중치를 메모리에서 읽어 곱해야 한다.** 이게 메모리 바운드(Memory-Bound) 워크로드. PIM이 가장 잘 맞는 영역.

### 6-3. 트랜스포머와 PIM 궁합

GPT·Llama·Claude 같은 LLM은 모두 **트랜스포머(Transformer)** 아키텍처. 핵심 연산:
1. **Attention**: Query × Key^T(행렬 곱) → Softmax → × Value(또 행렬 곱)
2. **FFN(Feed-Forward Network)**: 입력 × 가중치(행렬 곱) × 활성화 함수

전부 **행렬 곱**. PIM의 MAC 유닛이 가장 잘 처리.

#### 실측 사례 (SK하이닉스 발표, 2024)
- GPT-2 (1.5B 파라미터) 추론
- NVIDIA A100 GPU: 전력 400W, 처리량 X
- AiMX(AiM 기반 가속기): 전력 25W, 처리량 X+30%
- **전력 효율: 16배** (W당 처리량 기준)

### 6-4. Decode 단계가 PIM 천국

LLM 추론은 2단계.
1. **Prefill** (입력 전체 한 번에 처리) — 연산 바운드, GPU 우위
2. **Decode** (한 토큰씩 출력) — 메모리 바운드, **PIM 우위**

문제: 실제 사용자 경험(채팅·번역·코드 생성)에서 **Decode가 시간의 80~90%**. 즉 PIM이 잘하는 영역이 사용자 체감의 대부분.

> 💡 **숫자**: 사용자가 "긴 답변"을 받을수록 Decode 비중 증가. ChatGPT가 200 토큰 답할 때 Decode 95%, 50 토큰 답할 때 Decode 70%. PIM이 잘되면 **답변 속도가 2~3배** 빨라진다.

2. **HBM 추격 중인 약점이 PIM에서는 강점** — HBM 본체에서는 SK하이닉스에 밀리지만, **PIM 결합에서는 1년 이상 앞섬**.
3. **모바일 AP(엑시노스) + LPDDR-PIM 시너지** — 갤럭시 폰에 자체 통합 가능.

#### 핵심 약점
1. NVIDIA HBM4 인증 통과 여부 불확실 — 본체 HBM 못 팔면 PIM도 못 판다.
2. PIM 자체 매출 아직 $0 수준(2025) → 본업에 묻혀 보이지 않음.

#### 밸류에이션 기여
- 본업 시총 약 530조 원(2026.5 기준)
- HBM 옵션 가치 약 80조 원 (HBM 매출 EV/Sales 5배 가정)
- **PIM 옵션 가치 추가 10~30조 원** (Bull 시나리오)
- → 삼성 시총의 약 2~5%가 PIM 옵션

> 💡 **투자 포인트**: 삼성을 PIM 단독으로 사진 마라. **본업 + HBM + PIM 옵션의 합**으로 봐야 한다. PIM은 +α.

| 제품 | 종류 | 출시 | 상태 |
|---|---|---|---|
| GDDR6-AiM | GDDR-PIM | 2022 | 양산, 일부 고객 출하 |
| AiMX | GDDR6-AiM 기반 가속 카드 | 2023 | 고객 평가 |
| HBM-AiM | HBM3e/HBM4-AiM | 2026~2027 | 개발, 일부 샘플 |
| LPDDR-AiM | LPDDR-PIM | 2025~ | 검토 |

#### 핵심 강점
1. **HBM 점유율 60% 압도적 1위** → HBM-AiM 확장 시 자연스러운 시장 점유
2. **NVIDIA·AMD와 깊은 관계** → 차세대 GPU에 PIM 통합 시 우선 협력
3. **AiM 단독 가속기(AiMX)** → 자체 추론 가속기 사업 가능성

#### 핵심 약점
1. 메모리 단일 회사 → PIM 실패 시 충격 큼
2. 파운드리 없음 → 로직 부분 외주(TSMC 등) 필요, 마진 양보

#### 밸류에이션 기여
- 본업 시총 약 1,015조 원(2026.5)
- HBM 옵션 이미 시총에 반영
- **PIM 추가 옵션 약 20~50조 원** (Bull)
- → SK 시총의 약 2~5%가 PIM 옵션

### 9-4. 한계 — PIM이 못 하는 것

## 10. 밸류에이션 — 어떻게 평가하나 + 현재 위치

### 10-1. 평가 방법론(짧게)

PIM은 시장이 작고 초기 단계라 전통적 PER·EV/EBITDA 적용 어렵다. 3가지 접근.

#### 1) SOTP (Sum-of-the-Parts)
대형주(삼성·SK)는 본업 가치 + PIM 옵션 가치를 분리.
- 본업 = PER, EV/Sales 같은 전통 지표
- PIM 옵션 = 미래 시장 점유 × 매출 × Multiple

#### 2) Real Option Valuation
비상장·소형주(GSIT·Mythic 등)는 **블랙-숄즈 옵션 모델** 응용.
- 현재 매출은 거의 0, 미래에 큰 시장이 열릴 가능성
- 옵션 가치 = max(0, 시장 점유 × 시장 규모 - 비용)

#### 3) Comparable Multiple
유사 AI 가속기 회사(Cerebras·SambaNova·Graphcore)와 비교.

### 10-2. 삼성전자 PIM 옵션 가치

#### 가정 (Base 시나리오)
- 2030 PIM 시장 $20B
- 삼성 점유 30% = $6B 매출
- EV/Sales 5배(고성장 반도체) = $30B = **40조 원 옵션 가치**
- 현재 시총 530조 원 대비 약 **7.5%**

#### Bull / Bear
- Bull: 점유 40% × 시장 $30B × Multiple 7배 = $84B = **115조 원** (현 시총의 22%)
- Bear: 점유 20% × 시장 $8B × Multiple 3배 = $4.8B = **6.5조 원** (현 시총의 1%)

> 💡 **결론**: 삼성에 PIM은 본업 대비 1~22% 옵션. 시총 변동성 안에 묻혀 보이지 않을 가능성 높음. **PIM 단독으로 삼성을 사기보다는, 본업 + α로 봐야.**

### 10-3. SK하이닉스 PIM 옵션 가치

#### Base
- 2030 점유 35% × $20B = $7B 매출
- EV/Sales 5배 = $35B = **48조 원**
- 현재 시총 1,015조 원 대비 약 **4.7%**

#### Bull / Bear
- Bull: 45% × $30B × 6배 = $81B = **111조 원** (시총의 11%)
- Bear: 25% × $8B × 3배 = $6B = **8조 원** (시총의 0.8%)

> 💡 **시사점**: SK가 PIM에 더 적극적이지만, 본업 시총이 너무 커서 PIM 옵션 비중은 5% 내외.

### 10-4. GSI Technology — 옵션 가치 베팅

#### 현재 상태
- 시총 $150M
- 매출 $25M, 적자
- PIM 단독 비즈니스 → 큰 시장 열리면 폭발, 아니면 소멸

#### Bull / Base / Bear
| 시나리오 | 가정 | 매출 2030 | 시총 | 현재 대비 |
|---|---|---|---|---|
| Bull | APU가 정부·금융 표준 | $300M | $3B | +20배 |

## 부록 0. 추가 심층 — 한국 투자자가 더 알아야 할 것들

### 부록 0-1. PIM의 기술적 세부 — "왜 만들기 어려운가"

#### D램 공정 vs 로직 공정의 충돌

D램 셀과 로직(연산 회로)은 **완전히 다른 반도체 공정**으로 만든다.

| 항목 | D램 공정 | 로직 공정 |
|---|---|---|
| 트랜지스터 종류 | DRAM Cell Capacitor + Access Tr | FinFET, GAA |
| 노드 (2025) | 1a~1c nm급 (실제 12~14nm) | 3~5nm (TSMC), 4nm (삼성 LSI) |
| 클럭 속도 | 1~3 GHz | 3~5 GHz |
| 누설전류 | 매우 낮아야 함 (보존성) | 빠른 동작 위해 허용 |
| 마스크 수 | 약 25~30장 | 약 80~100장 |
| 비용/wafer | $2,000~$3,000 | $15,000~$20,000 |

**문제**: PIM은 두 공정을 한 칩에 넣어야 한다. 셀 공정으로 로직을 만들면 **느리고 크고**, 로직 공정으로 셀을 만들면 **누설전류로 데이터 사라진다**.

**해결책 3가지**:
1. **타협 공정** — 1.5세대 D램 공정에 로직 일부 (삼성 HBM-PIM)
2. **3D 적층** — D램 다이 + 로직 다이를 별도 제조 후 TSV로 결합 (HBM 베이스 다이 활용)
3. **모놀리식 3D** — 한 wafer에 D램층 + 로직층을 순차 적층 (R&D 단계, 2030년 이후)

> 💡 **시사점**: 삼성이 메모리·파운드리 동시 보유 = 두 공정 모두 자체 가능 = 최강. SK는 메모리만 → TSMC와 협력 필요.

#### TSV(Through-Silicon Via)의 역할

HBM-PIM은 **TSV**(실리콘 관통 비아) = 칩에 미세한 구멍을 뚫어 위·아래 다이를 전기로 연결하는 기술이 핵심.

- HBM3e 1 스택 = 12장 다이 적층, **TSV 1024개**
- HBM-PIM은 TSV로 **연산 결과를 위·아래로 전달**
- TSV 수율이 PIM 수율의 80% 결정

#### Bank-Level Parallelism

D램은 내부에 **수백 개 뱅크(Bank)**로 나뉨. 각 뱅크가 독립적으로 동작 가능.

PIM의 핵심: **각 뱅크 옆에 작은 MAC 유닛을 배치**해서 동시에 수백 개 연산 수행.
- 일반 D램: 한 번에 1개 뱅크만 활성
- HBM-PIM: 16~32개 뱅크 동시 연산 = **병렬도 16~32배**

### 부록 0-9. 자주 받는 질문 FAQ

**Q1. PIM이 GPU를 대체하나?**
A. **아니다**. PIM은 추론(특히 Decode) 가속, GPU는 학습 강자. **공존**할 가능성이 높다. NVIDIA가 GPU 안에 HBM-PIM 통합하면 둘 다 팔리는 구조.

**Q2. 삼성·SK 중 누가 PIM에서 더 유리한가?**
A. **현재 삼성 1년 앞섬**(HBM-PIM 양산 1호). 그러나 **SK가 NVIDIA·AMD와 깊은 관계** → 양산 채택은 SK가 빠를 수도. 둘 다 가지고 있는 게 안전.

**Q3. 직접 살 수 있는 PIM 단독 종목이 있나?**
A. **GSI Technology(GSIT)** 단 하나. 시총 $150M 소형주, 변동성 큼. 1~3% 포지션 권장.

**Q4. PIM 매출이 언제부터 의미 있게 잡히나?**
A. **2027~2028**부터. 2026까지는 R&D + 일부 샘플 매출 수준. 본격 양산은 NVIDIA 채택 시점에 달림.

**Q5. PIM 실패 시나리오는?**
A. NVIDIA가 자체 HBM 베이스 다이에 연산 통합 → 별도 PIM 시장 흡수. 한국 회사들은 본업 + 약간 수익 정도.

**Q6. 한국 정부 지원은?**
A. K-반도체 전략 안에 PIM 포함, 약 1조 원 R&D 지원(2024~2028). 미국 CHIPS Act는 직접 PIM 항목 없음.

**Q7. PIM 칩이 양산되면 D램 가격이 떨어지나?**
A. **반대로 오를 가능성**. PIM은 일반 D램보다 비싸게 팔리고(2~5배), 메모리 회사 마진 개선.

**Q8. AI 거품이 꺼지면 PIM도 끝나나?**
A. **AI 추론은 거품이 아니라 인프라**. ChatGPT 사용량은 계속 증가 → 추론 비용 절감 수요는 영구. PIM은 살아남는다.
