## 1️⃣ 반도체의 큰 그림 — 메모리 vs 비메모리

반도체는 크게 두 가지로 나뉩니다.

### 메모리 반도체 (Memory)
- **데이터를 저장**하는 칩
- DRAM, NAND Flash, HBM, SRAM 등
- **표준품(Commodity)** 성격 → 가격 변동성 큼, 사이클 강함
- 한국이 세계 1·2위 (Samsung·SK하이닉스)

### 비메모리 반도체 (Non-Memory / Logic)
- **연산·제어**를 하는 칩
- CPU, GPU, AP, NPU, MCU, ASIC, Power IC 등
- **맞춤형(Customized)** 성격 → 마진 안정적
- 미국·대만 주도 (NVIDIA, AMD, Intel, Qualcomm, TSMC)

### 왜 구분이 중요한가?
| 축 | 메모리 | 비메모리 |
|---|---|---|
| 글로벌 시장 규모 | ~$180B (2024) | **~$450B (2024)** |
| 비중 | **28%** | 72% |
| 사이클 | 3~4년 (호황·불황 뚜렷) | 상대적으로 안정 |
| 대표 기업 | Samsung, SK하이닉스, Micron | NVIDIA, Intel, TSMC, Qualcomm |

> 📌 **한국은 "메모리 강국", "비메모리 약국"** — 비메모리 세계 점유율이 3% 수준에 그침. 이게 윤석열 정부의 "시스템반도체 K-Chips Act"의 배경.

## 2️⃣ 메모리 반도체 — DRAM·NAND·HBM 기본 개념

### DRAM (Dynamic Random Access Memory)
- **빠르지만 전원 꺼지면 사라짐** (휘발성)
- 컴퓨터의 "단기 기억" — CPU가 지금 쓰는 데이터
- 구조: 1 셀 = **1 트랜지스터 + 1 캐패시터** → 주기적 재충전(refresh) 필요
- 속도: 매우 빠름 (ns 단위)
- 용도: PC RAM, 서버 RAM, 스마트폰 RAM

### NAND Flash
- **전원 꺼져도 유지됨** (비휘발성)
- 컴퓨터의 "장기 기억" — 파일 저장
- 구조: 1 셀에 여러 비트 저장 가능 (TLC 3bit, QLC 4bit)
- 속도: DRAM보다 100~1,000배 느림
- 용도: SSD, USB, 스마트폰 스토리지

### HBM (High Bandwidth Memory)
- **DRAM을 수직으로 쌓은 것** (3D 적층)
- AI GPU 옆에 바로 붙여서 초고속 통신
- **"초고속 DRAM" = HBM** 으로 이해하면 됨

### 셋의 관계를 비유하면
- **DRAM** = 책상 위 펼쳐놓은 노트 (빠른 접근, 퇴근하면 사라짐)
- **NAND** = 서랍 속 책 (느리지만 영구 보관)
- **HBM** = 책상에 책 수십 권을 계단처럼 쌓아놓은 것 (공간 절약 + 초고속)

### DRAM vs NAND 비교
| 축 | DRAM | NAND |
|---|---|---|
| 휘발성 | O (전원 꺼지면 사라짐) | X (영구 보존) |
| 속도 | 매우 빠름 (ns) | 느림 (μs~ms) |
| 가격 | 비쌈 ($/GB) | 저렴 |
| 집적도 | 낮음 (2D) | 높음 (3D 400단+) |
| 플레이어 | 3사 과점 | 5사 + YMTC |
| 진입 장벽 | 매우 높음 | 상대적 낮음 |

## 3️⃣ DRAM 심화 — 서버·PC·모바일·HBM

### DRAM 종류 (용도별)

| 종류 | 전체 이름 | 주요 용도 | 특징 |
|---|---|---|---|
| **DDR** | Double Data Rate | PC, 서버 | 현재 DDR5 주류, DDR6 개발 중 |
| **LPDDR** | Low Power DDR | 스마트폰 | 저전력, LPDDR5X → LPDDR6 |
| **GDDR** | Graphics DDR | GPU (게임용) | 고속, GDDR6X·GDDR7 |
| **HBM** | High Bandwidth Memory | AI GPU·서버 | 3D 적층, 초고속 |

### 서버 DRAM이 왜 중요한가?
- 비중이 가장 빠르게 커지는 세그먼트
- Intel Granite Rapids·AMD EPYC Turin 등 차세대 서버 CPU → **DDR5 필수**
- 서버 DRAM ASP가 PC·모바일 대비 2~3배

### 2026년 DRAM 빅 이슈
- **HBM이 전체 DRAM wafer의 40%+를 잠식** → 일반 DRAM 부족
- DDR5 서버 가격 **4Q25 대비 1Q26 +60%** 폭등
- PC·모바일 OEM 다운스펙 시작 (1TB → 512GB)
- **중국 CXMT**가 DDR5 8,000 Mbps·24Gb 발표, HP·Dell·ASUS 평가

## 4️⃣ NAND 심화 — 3D NAND·TLC/QLC·eSSD

### 3D NAND란?
- 과거: 2D (평면에 셀 배치)
- 현재: **3D** (빌딩처럼 수직으로 쌓음)
- 2026년 현재 **300단 이상** 경쟁: SK하이닉스 321단, Kioxia 332단, Samsung 286단
- **Layer가 많을수록 같은 면적에 더 많은 저장 용량**

### MLC → TLC → QLC → PLC

| 타입 | 셀당 저장 | 밀도 | 내구성 | 용도 |
|---|---|---|---|---|
| SLC (1bit) | 1 | 기준 | 10만+ cycles | 구형, 고신뢰 |
| MLC (2bit) | 2 | +100% | 1~3만 | 구형 |
| **TLC** (3bit) | 3 | +200% | 1,500~3,000 | **주력** (cSSD, UFS) |
| **QLC** (4bit) | 4 | +267% | ~1,000 | **eSSD 대용량** |
| PLC (5bit) | 5 | +333% | <500 | 아카이벌 (실험) |

→ 한 셀에 많이 저장할수록 싸지지만 내구성↓. **AI 데이터센터의 QLC eSSD** (122TB, 245TB)가 새로운 성장축.

### eSSD의 부상 — AI 시대의 핵심
- AI inference 스택: **HBM** (모델 weight) → **DDR5** (working memory) → **NAND eSSD** (KV cache, vector DB, RAG corpus)
- NVIDIA가 **ICMS (Inference Context Memory Storage)** 플랫폼으로 공식 편입
- Vera Rubin: **GPU당 16TB eSSD**, 랙당 2.3PB

### NAND 5대 + 중국
| 기업 | M/S (2025 Q3) |
|---|---|
| Samsung | 32.3% |
| SK하이닉스 + Solidigm | 19.3% |
| Kioxia | 15.3% |
| Micron | ~13% |
| SanDisk | 12.4% |
| YMTC (중국) | ~11.8% |

<a id="5"></a>
## 5️⃣ HBM 심화 — AI 시대의 주인공

### HBM이란?
- **DRAM 여러 개를 수직으로 쌓고 (stack)**, **실리콘을 관통하는 구멍 (TSV)** 으로 전기 신호를 보내는 DRAM
- 기존 DRAM 대비 **대역폭 10배+, 전력 효율 3배+**

### HBM 세대
| 세대 | 출시 | 대역폭 | 용량 (GB) | 특징 |
|---|---|---|---|---|
| HBM1 | 2015 | 128 GB/s | 4 | 초기 |
| HBM2 | 2016 | 256 GB/s | 8 | 널리 채택 |
| HBM2E | 2019 | 460 GB/s | 16 | |
| HBM3 | 2022 | 819 GB/s | 24 | H100 채택 |
| **HBM3E** | 2024 | 1.2 TB/s | 36~48 | B200 채택, **현재 주력** |
| **HBM4** | 2026 | 2.0 TB/s+ | 48~64 | Rubin·MI400 채택 |
| HBM4E | 2027+ | 더 높음 | 더 큼 | 로드맵 단계 |

### HBM이 왜 공급 부족인가?
- **HBM 1GB = 표준 DRAM 1GB의 wafer 면적 3배** (TSV, 다층 stack, KGD 선별 손실)
- 즉, 같은 fab에서 HBM을 만들면 **일반 DRAM을 3배 덜 만들게 됨**
- 2026년 전체 DRAM wafer의 **40%+ 가 HBM에 잠식**

### HBM 3사 구도
| 기업 | 2025~26 M/S | 주요 고객 |
|---|---|---|
| **SK하이닉스** | **53~62%** | NVIDIA ~90%, AMD·Google |
| **Samsung** | 17~35% (HBM3E 12-Hi 승인 후 회복) | NVIDIA HBM4 30%+, Google TPU 60%+ |
| **Micron** | 11~25% | NVIDIA H200/B200, AMD MI350 |

### TSV·CoWoS가 왜 병목인가?
- HBM은 **TSMC의 CoWoS** 패키징에서 GPU와 합쳐짐
- CoWoS capacity가 부족 → GPU·HBM 생산량을 **패키징 단계에서 cap**
- TSMC 2025~2026 CoWoS 월 75~80K → **2026 말 120~130K** 목표

## 6️⃣ 비메모리(Logic) — CPU·GPU·AP·ASIC

### 주요 종류

| 종류 | 역할 | 대표 기업 |
|---|---|---|
| **CPU** (Central Processing Unit) | 범용 연산, "머리" | Intel (x86), AMD, Apple (M1~M4), ARM |
| **GPU** (Graphics Processing Unit) | 병렬 연산, **AI 학습/추론** | NVIDIA, AMD |
| **AP** (Application Processor) | 스마트폰용 SoC | Qualcomm (Snapdragon), Apple (A), Samsung (Exynos), MediaTek (Dimensity) |
| **NPU** (Neural Processing Unit) | AI 전용 | Apple, Samsung, Qualcomm에 내장 |
| **ASIC** (Application-Specific IC) | 특정 작업 전용 | Google TPU, AWS Trainium, Meta MTIA |
| **MCU** (Microcontroller) | 가전·IoT | STMicro, NXP, Infineon, Renesas |
| **Power IC** | 전력 관리 | TI, Infineon, Navitas, Onsemi |

### Fabless vs IDM vs Foundry vs OSAT

| 모델 | 설명 | 예시 |
|---|---|---|
| **Fabless** | 설계만 함 (제조 외주) | NVIDIA, AMD, Qualcomm, Apple, Broadcom |
| **IDM** (Integrated Device Manufacturer) | 설계 + 제조 + 패키징 모두 | Intel, Samsung (메모리), TI, Micron |
| **Foundry** | 제조만 (설계 안 함) | TSMC, UMC, GlobalFoundries, SMIC |
| **OSAT** (Outsourced Assembly & Test) | 후공정·패키징 전문 | ASE, Amkor, JCET |

### AI 칩 전쟁
- **NVIDIA 독주** — H100 → B200/B300 → Rubin/Rubin Ultra
- AMD 추격 — MI300 → MI350 → MI400
- **하이퍼스케일러 자체 칩**:
  - Google TPU (Ironwood = v7)
  - AWS Trainium 3, Inferentia
  - Meta MTIA 300/400
  - Microsoft Maia 200

## 7️⃣ 파운드리 — TSMC·Samsung·Intel

### Foundry란?
- Fabless 고객의 설계를 받아 **칩을 제조**하는 회사
- 공정 기술 (nm) + 생산 능력 + 수율이 핵심 경쟁력

### 글로벌 Foundry 시장 (2025)
| 기업 | M/S | 특징 |
|---|---|---|
| **TSMC** | **~60%** | 독주, 2~3nm 선도, CoWoS 패키징 독점 |
| Samsung Foundry | ~10% | GAA 3nm 선도 but 수율 열위 |
| SMIC (중국) | ~5% | 7nm 도달, 美 제재 |
| UMC (대만) | ~5% | 성숙 공정 |
| GlobalFoundries | ~5% | 성숙 공정 + 미국 공장 |
| Intel Foundry | 신규 | 18A (2025), 14A (2027) 로드맵 |

### 미세공정 (Process Node)
- nm = nanometer (10억분의 1 미터)
- 작을수록 **같은 면적에 더 많은 트랜지스터** → 성능↑ 전력↓
- 현재 주력: **3nm** (TSMC N3)
- 2025~2026: **2nm** (TSMC N2, Samsung SF2)
- 2027~2028: **1.4nm / A14** (TSMC)
- Intel: 18A(1.8nm 상당), 14A(1.4nm 상당)

### GAA vs FinFET
- **FinFET** (14nm~3nm 주류): 트랜지스터가 "지느러미" 모양
- **GAA (Gate-All-Around)**: 3nm~ 사용, 게이트가 채널을 완전히 감쌈 → 성능·전력↑
- Samsung이 GAA 3nm를 먼저 도입했으나 **수율 열위로 시장 주도권 상실**

## 9️⃣ EUV·High-NA EUV — 미세공정의 핵심

### EUV (Extreme Ultraviolet Lithography)
- 극자외선 파장 **13.5 nm** 로 패턴을 그리는 기술
- **ASML이 세계 유일 제조사**
- 7nm 이하 공정에 필수
- 한 대 가격 **$150~200M** (2,000억 원+)

### High-NA EUV (EXE:5200 시리즈)
- "Numerical Aperture"가 0.33 → **0.55** 로 커짐
- 더 작은 패턴 그릴 수 있음 → 2nm 이하 공정에 필수
- 한 대 **$350~400M**
- Intel이 first customer, SK하이닉스 2025.9 DRAM fab 설치, Samsung 2025 말~2026 상반기

### 왜 ASML이 독점인가?
- EUV 광원은 **액체 주석 방울에 레이저를 초당 5만 번 쏴서** 만드는 극도로 복잡한 기술
- 미러 시스템이 **Zeiss (독일)** 단독 공급
- 30년+ 축적된 know-how, 경쟁자 진입 사실상 불가능

### 미국의 대중국 EUV 제재
- 2019년부터 네덜란드 정부가 ASML의 중국 수출 차단
- 중국은 SMEE 등 자체 EUV 개발 시도 중이나 **최소 10~15년 격차**
- **중국이 7nm 이하를 못 가는 구조적 원인**

<a id="10"></a>
## 🔟 후공정(OSAT)·패키징 — CoWoS가 왜 중요한가

### 전공정 vs 후공정

| 단계 | 내용 | 주요 기업 |
|---|---|---|
| **전공정** (Front-end) | 웨이퍼에 트랜지스터·배선 형성 | TSMC, Samsung |
| **후공정** (Back-end) | 칩 절단, 패키징, 테스트 | ASE, Amkor, TSMC (CoWoS) |

### CoWoS (Chip-on-Wafer-on-Substrate)
- **TSMC의 고급 패키징 기술**
- GPU + HBM을 한 기판 위에 나란히 배치 → 2.5D 패키징
- NVIDIA H100·B200·Rubin **전부 CoWoS 기반**
- **CoWoS capacity 부족 = AI 칩 공급 부족의 본질적 원인**

### CoWoS-L vs CoWoS-S
- **CoWoS-S** (Si interposer) — 기존, 작은 크기
- **CoWoS-L** (LSI + RDL) — 신형, 더 큰 다이 조합 가능
- 2026년 Rubin Ultra·B300은 CoWoS-L 필수

### SoIC (System on Integrated Chips)
- TSMC의 3D 패키징 (칩을 수직 적층)
- AMD Ryzen X3D (3D V-Cache)가 대표
- 미래 AI 칩은 SoIC + CoWoS 결합으로 진화 예상

## 1️⃣1️⃣ 반도체 사이클 — 왜 메모리는 3~4년 주기인가

### 메모리 사이클의 본질
1. **수요 호황** → 가격 ↑
2. 모든 기업이 공격적 증설 → **공급 과잉**
3. 가격 폭락, 재고 쌓임 → **감산**
4. 수요 회복 → 다시 호황

### 최근 사이클
| 구간 | 상태 |
|---|---|
| 2017~2018 | **역대급 슈퍼사이클** (Samsung 사상 최대 이익) |
| 2019 | 크래시 (무역전쟁, 공급 과잉) |
| 2020~2021 | 코로나 수혜 회복 |
| 2022~2023 | **크래시** (PC 수요 붕괴, 재고) |
| 2024 | 바닥 통과 |
| **2025~2028** | **AI 슈퍼사이클** (HBM 주도, 구조적 변화) |

### 2017~18 vs 2025~28 차이 (중요)
| 축 | 2017~18 | 2025~28 |
|---|---|---|
| 수요 주체 | 범용 서버 | **AI 학습·추론 구조적 수요** |
| 가격 상승 | DRAM 2년 +80% | 누적 **+275~300%** |
| 길이 | ~9분기 | **2028년까지 연장** 전망 |
| 증설 반응 | 공격 증설 → 2019 크래시 | **"nobody's scaling up"** 보수적 |
| 구조적 wafer 잠식 | 미미 | **HBM 40%+ 잠식** (영구적) |

## 이 산업의 밸류에이션은 어떻게 평가하나?

사과 1kg에 얼마인지 묻듯, 주식도 "얼마짜리 이익을 얼마에 사는지"를 잰다. 반도체는 사이클성이 워낙 강해 호황 P/E와 불황 P/E가 정반대 의미를 가진다.

이 산업에서 주로 보는 잣대는 **P/E, EV/EBITDA, P/B-ROE** 세 가지다.

- **P/E (주가수익비율):** 가장 친숙한 잣대지만 사이클 정점에선 P/E가 낮아도(이익 정점) 위험 신호, 바닥에선 높아도(이익 바닥) 매수 타이밍일 수 있다.
- **EV/EBITDA:** 거대 Capex로 감가상각이 큰 산업이라 EBITDA 기준이 더 안정적. 12~15배가 평년, 8배 이하면 사이클 바닥 시그널.
- **P/B-ROE:** 메모리는 P/B를 가장 많이 본다. 같은 P/B라도 ROE가 높을수록 정당화 — "P/B 2배인데 ROE 25%"는 비싸 보여도 합리적.

> **주의할 점:** "P/E가 낮으니 싸다"는 사이클 정점에서 가장 위험한 함정. 이익이 다음 분기부터 절반 날 수 있다.

**역사적 위치 감각:** 메모리(삼성전자·하이닉스)는 P/B 1배 근처가 사이클 바닥, 2~3배가 정점. 파운드리(TSMC)는 P/E 18~25배 박스에서 움직이고, AI 슈퍼사이클 진입 시 30배 위로 리레이팅. 2026년 현재는 HBM 호황에 P/B가 정점 구간.

이 잣대는 종목 분석 리포트의 매트릭스에서 구체 수치로 확인.
